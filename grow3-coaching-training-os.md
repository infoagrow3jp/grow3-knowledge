# 第1部　中核定義（二層構造）

コーチングの定義は「目的の定義」と「構造の定義」の二層で持つ。受講者向けの説明には目的の定義、ファシリテーター自身の設計理解には構造の定義を使う。

**目的の定義（受講者向け・外向き）**
コーチングとは、相手が今どこまで見えているかを確認しながら、自分で考え、判断し、行動できる範囲を少しずつ広げていくための対話である。

言い換え：部下を変える技術ではなく、部下が自分で考え、動ける状態を一緒につくる関わり方。

受講者への説明文（研修冒頭で使える形）：
「コーチングは、部下に答えを出させるテクニックではありません。まして、質問だけで相手を思い通りに動かす技術でもありません。コーチングとは、相手が今の状況を整理し、どうなりたいかを言葉にし、現実とのギャップに気づき、次に何をするかを自分で決められるようにする会話です。」

**構造の定義（ファシリテーター向け・設計理解）**
コーチングとは、方法論（どうやるか）は相手に委ねるが、論点（何について考えるか＝現状・目標・ギャップ・選択肢・次の一歩）はコーチが握る、という二層構造の対話である。

- コーチが完全に無色透明で臨むセッションは機能しない。論点の地図（会話が今どのフェーズにいるか）をコーチが把握しているから、対話が発散せずに前へ進む。
- 境界線：論点の順序・種類はコーチが決めてよいが、論点に対する答え・評価はコーチが決めない。「目標は何ですか」は論点の提示（OK）、「その目標は現実的ではない」は答えへの評価（コーチングから逸脱）。
- この境界線は、実践ワークの観察役の観察基準にもなる：①コーチが論点を適切に提示できていたか、②相手の答えに評価を挟んでいなかったか。

**記憶・学習の観点からの裏づけ**
人は情報をそのまま保存するのではなく、既存の経験・意味・感情と結びつけて（意味づけて）覚える。だから上司が一方的にWHATを渡すだけでは判断基準が育たず、本人が自分の言葉で話し・整理し・決めることで記憶にも行動にも残る。コーチングは、この意味づけと行動化を本人の中で起こすための対話である。[→F:経験学習サイクル][→F:メタ認知]

受講者に伝えるときの言い換え（「連想記憶」等の用語は使わない）：
- 人は、意味づけできたことほど覚えやすい
- 自分で話したことほど気づきやすい
- 自分で決めたことほど動きやすい

注：ラーニングピラミッドの数値（講義5%等）は実証根拠が乏しく引用しない。「受け身より、思い出す・使う・説明する・対話する方が学びやすい」という方向感のみ使う。

# 第2部　ティーチングとコーチングの整理

**基本の対比（To Be → To Do 型）**
- ティーチング：To Do を渡す関わり。「何をすればいいか」を教える。
- コーチング：To Be（どうなりたいか・どうありたいか）を一緒に描き、そこから本人の To Do を引き出す関わり。

**WHY→HOW→WHAT 型**
- 普通のティーチング：WHAT→HOW→WHY の順になりやすい（作業はできるが意味のネットワークに入らない）
- 良いティーチング：WHY→HOW→WHAT で教える
- コーチング：相手が WHY→HOW→WHAT で考えられるように促す

**使い分けの原則**
コーチングは万能ではない。知識・経験が足りない人には教える（ティーチング）。経験があるのに考えが整理できていない人には引き出す（コーチング）。この使い分けこそが管理職のスキル。[→F:リーダーシップ（SL理論）]

「行動が起こせない」と「やる気がない」は違う。部下が動かないとき、実際には：状況が整理できていない／何が問題かわかっていない／どこから始めればいいかわからない／自分にできる感覚がない／失敗したときの不安がある／上司に言っていいかわからない──ことが多い。コーチングはやる気の注入ではなく、動けない理由を一緒にほどく会話。

# 第3部　再利用フレーム集（7つの型）

コーチング研修に限らず、管理職研修・OJT研修・1on1研修・フィードバック研修に転用可能。

**1. To Be → To Do 引き出し型**
第2部参照。研修冒頭の説明、募集文の骨格に使える。

**2. WHY→HOW→WHAT 型**
第2部参照。建設・製造・医療介護・営業など業種を問わず使える。

**3. 表面張力型（Grow3独自表現）**
相手が今どこまで見えているかを見極めながら、考えがちぎれない範囲で、少し先へ引き上げる。
- 引っ張りすぎると苦しくなる。引っ張らなさすぎると成長しない。
- だから、相手が自分で考え続けられるギリギリに問いを置く。
コーチングを「優しい会話」ではなく育成の技術として説明できる。研修中のキーフレーズ・ブログ転用可。

**4. 現状→理想→ギャップ→選択肢→確認→承認 型**
質問例シートの骨格。第4部参照。1on1研修・管理職面談研修にそのまま転用可。

**5. 「育てないから忙しい」型（管理職向けの動機づけ）**
部下を育てる時間がないから忙しい。でも、育てないからずっと忙しい。
→ コーチングは、上司が抱え続けないために、部下の考える力と動く力を育てる関わり方である。
コーチングを「人に優しくする話」ではなく「管理職自身の時間をつくる話」にできる。

**6. 主体性の前処理型**
主体性は本人のやる気だけで決まらない。以下の状態があって初めて発揮される：
- 自分で考えてよい／自分にもできそうだ／相談しても大丈夫だ／失敗しても一緒に振り返れる／次に何をすればいいか見えている
→ コーチングは、主体性を求める前に、主体性が出る状態をつくる関わり。
接続：この「状態をつくる」発想は SSR理論（人材力・組織力・関係力の連動で成果が出る構造）と同型。個人のやる気に帰属させず、構造・関係の側を整える点で一致する。[→F:SSR理論][→F:心理的安全性][→F:選択理論心理学]

**7. 学習設計型（講義→ミニ練習→ロールプレイ→観察FB→実践計画）**
第5部参照。ハラスメント研修・任せ方研修・フィードバック研修にも転用可。

**中核フレーズ（一本にまとめる場合）**
コーチングとは、相手が今どこまで見えているかを確認しながら、自分で考え、判断し、行動できる範囲を少しずつ広げていくための対話である。──これを中心に、To Be→To Do／WHY→HOW→WHAT／表面張力／育てないから忙しい／現状→理想→ギャップ→次の一歩、を周辺に配置する。

# 第4部　質問の型

**質問例シートの5フェーズ（＋クロージング）**
コーチング会話の論点の順序。順序はシートが決め、中身は相手の言葉で埋まる（＝第1部の二層構造の実装）。

1. 現状把握・明確化（5問以上）：「今はどんな状態ですか？」「〜というと？」「他には？」「10点満点で現在何点？」
2. 目標明確化・理想を描く（2問以上）：「どんな状態になったら最高にうれしいですか？」「実現したときどんな気持ちに？」「現状○点から何点にしたい？」
3. ギャップの明確化（必ず質問）：「理想と現実の間にあるギャップとは？」
4. 選択肢の模索（4問以上）：「出来るかどうか別にしてどんなアイデアが？」「やめたほうがいいことは？」「協力者は？」「何からはじめますか？」
5. 確認（1問以上）：「今週（今月）は何をしますか？」「次回どんな報告をいただけますか？」
6. 最後に（2つとも）：相手から受ける印象や強みを伝える／「あなたならできます」等のメッセージを贈る

**デモコーチングの時間配分（15分の場合）**
現状把握5分 → 目標3分 → ギャップ1分 → 選択肢4分 → 確認1分 → 承認1分
- デモで一番見せるのは現状把握の深掘り（受講者が実践で最も失敗しやすいのが、現状把握を1問で切り上げること）
- ①で点数を聞いておくと②の「何点にしたい？」が自然につながり、数値の対比でギャップが浮かぶ構造を見せられる
- ③はクライアントに自分から言語化させる（コーチが分析しない）
- ④は「他には？」で3つ以上出させる粘りを見せてから収束
- 広げる質問（他には？）と絞る質問（何から始めますか？）の使い分けを見せることが質問スキル講義との接続点になる

**問いのスタンス（エドガー・シャインの4つの問いかけ）**
同じ質問文でも、発するスタンスで別物になる。「今の状態はどうですか？」を、謙虚なスタンス（本当に知りたくて聞く）で発するのと、診断的スタンス（答えを探りながら聞く）で発するのでは、相手の受け取り方が変わる。
- 謙虚な問いかけ（Humble Inquiry）：相手主導・開いている。純粋に知りたくて聞く。コーチングが目指す位置。
- 診断的な問いかけ：相手の思考を方向づける問い。
- 対決的な問いかけ：自分の見解を提示しながら問う。
- プロセス指向の問いかけ：「今ここ」の対話自体に焦点を当てる。
研修への組み込み方：質問の型を教えた後、「同じ質問でもスタンスで印象が変わる」体感ワーク（同じ一文を両スタンスでペアに言ってみる）を3〜5分挟む。コーチが無自覚にティーチングに寄るとき、言葉は質問形でもスタンスが診断的・対決的に寄っている。※2026年版では未実施。2027年版の検討材料。

# 第5部　コーチング研修の学習設計

**5段階の学習設計（2026年版で確立）**
講義 → ミニ練習 → ロールプレイ（本番形式） → 観察者フィードバック → 現場実践計画

- ミニ練習：本番ロープレの前に「型を試す小さな成功体験」を作る。質問が苦手な人がシートを読むだけで終わる事故を防ぐ。
- ロールプレイ：3人1組（コーチ・クライアント・オブザーバー）で全員が全役を体験。クライアント役は実際の仕事の悩みを話す（作り話より本物）。
- 観察者FB：コーチ・クライアント役だけでは気づけない「聴き方・問いの効き方」を第三者視点から言語化。観察基準は第1部の境界線（論点提示できていたか／答えに評価を挟んでいなかったか）。
- 現場実践計画：「学んで終わり」を防ぐ。相手・日時・使うスキル・最初の問いかけの4点を記入しペアで宣言。抽象的な感想（頑張って傾聴する等）で終わらせない。

**研修全体の一本線（章構成の論理）**
- 2章（なぜ必要か）：上司が答えを出すだけでは限界がある背景を自分事化
- 3章（何か）：教える場面と引き出す場面を分ける（使い分けの視点）
- 4章（どう関わるか）：承認＝話していい状態をつくる／傾聴＝考えを出せる場をつくる／質問＝現状・理想・ギャップ・選択肢を整理する／FB＝見えていない現在地を返す──4スキルは別々ではなく、相手が自分で考えるための補助線として役割が違うだけ
- 5章（どこで使うか）：今日の学びを誰とのどんな会話で使うか決めて帰る（目標管理研修に寄せない）

**各章振り返りの問い（章ごとに具体化する）**
全章共通の抽象的な問い（印象に残ったことは？）は形骸化する。Q1のみ章ごとに変え、Q2（変化）Q3（行動）は行動転移のため固定。
- 2章Q1：自分のチームで主体性が発揮されていないと感じる場面を1つ書いてください
- 3章Q1：コーチングとティーチング、自分は今どちらに偏っていると思いますか？
- 4章Q1：承認・傾聴・質問・FBのうち、今日一番手ごたえがあったスキルはどれですか？また、その理由も書いてみてください

# 第6部　2026年版改定の記録と2027年版方向性

**2026年版改定の要点（スコア52点→82点）**
- 削除・圧縮：3原則4枚→1枚統合／コーチの3視点・ジョハリ・トリプルループを配付資料化／第5章の目標管理系（カスケード・SMART・エンゲージメント）を「1on1が機能する前提」に絞り深掘りしない
