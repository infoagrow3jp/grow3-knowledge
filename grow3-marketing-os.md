## 1. ブランドの核

### ブランドの中心メッセージ

***経営者の想いを、社員が動ける仕組みに変える。——人を動かすのではなく、人が動きたくなる環境をつくる。***

### 一言ポジション

**利益を生み出し、納得して分かち合える組織をつくる、中小企業の組織づくりパートナー。**

### 顧客への約束

経営者の想いを「社員に伝わる言葉」と「現場で動ける仕組み」に変え、社員が自分の役割と評価に納得しながら動ける状態をつくります。人を責めるのではなく、仕組みと対話を整えることで、離職・育成・評価・賃金のモヤモヤを解消していきます。

### 公正・納得に関する考え方

グロウスリーが考える「公正・納得」は、全員に同じように配ることではありません。

会社が何に報いるのかを明らかにし、その方針が事業への貢献、お客様への価値、チームへの貢献とつながっていることを重視します。評価や賃金は、社員に不満を持たせないためだけのものではなく、会社が大切にしたい行動を伝える仕組みです。事業に貢献している人を堂々と評価できない状態は、本当の意味で公正とは言えません。大切なのは、誰にも差をつけないことではなく、何に報いるのかを明らかにすることです。

### 一言ポジションの背景にある経営論理

*グロウスリーが目指すのは、単に「社員にやさしい会社」をつくることではありません。利益が出なければ、賃上げも人材育成も継続できません。一方で、利益の分配に納得感がなければ、社員は成長や貢献に前向きになれません。だからこそ、利益を生み出すための役割・行動・成長の仕組みと、利益を納得して分かち合うための評価・賃金の仕組みを一体で設計します。*

### ミッション・ビジョン・バリュー

| | |
|---|---|
| **ミッション** | 経営者の想いを、社員が動ける仕組みに変える |
| **ビジョン** | 社員も経営者も、月曜日が待ち遠しくなる組織が当たり前の社会をつくる |
| **バリュー** | 対話から始める／仕組みで支える／現場に根づくまで伴走する |

## 3. 理想顧客・NG顧客

### 対象となる企業

| | |
|---|---|
| **企業規模** | 社員数30〜100名程度の中小企業 |
| **意思決定者** | 中小企業の経営者・社長、人事・総務担当者 |
| **地域** | 岡山県を中心とした西日本の中小企業（オンライン・研修派遣により広域対応も可能） |

### 理想の経営者像

ビジョンや人への想いを重視しながらも、最終的には経営として筋が通るかを大切にする経営者。「この人なら信頼できる」という感覚と「言っていることに根拠がある」という納得感の両方が揃ったときに動くタイプ。社員思いで、数字にも誠実に向き合おうとしている。

### 理想顧客の言葉

- 社員のことを考えて、給与も上げた。面談もした。研修も入れた。それでも、なぜか人が辞めていく。
- 社員が悪いとは思いたくない。でも、このままでは会社が前に進まない。
- 評価や給与の話になると、なんとなく気まずい。基準を聞かれると答えに詰まる。
- 経営者の想いはある。社員も悪い人ではない。でも、共通言語がなく、善意が空回りしている。
- 人事・組織の悩みを誰にも相談できない。顧問税理士は数字の話はしてくれるが、組織の話は専門外だ。

### ✅ 相性が良い顧客

- ビジョンはあるが組織がついてきていないと感じている経営者
- 社員を責めたくないが、このままではいけないと感じている経営者
- 感覚ではなく、納得できる仕組みで組織を整えたい経営者
- 経営者自身も対話と仕組みづくりに向き合う意思がある

### ❌ NG顧客・支援対象外

- 社員を一方的に管理・支配するための制度づくりを求める会社
- 評価制度を賃下げや退職誘導の道具として使いたい会社
- 経営者自身が対話や仕組みづくりに向き合う意思をまったく持っていない会社
- 短期間で社員を思い通りに動かす魔法の手法だけを求める会社
- 問題の原因を「問題社員」に求め、社員を排除することを主目的とする会社
- すべてを丸投げしたい会社

*ただし、「合わない社員がいる」「退職してほしい社員がいる」という経営者の本音そのものは否定しません。その感情を、感情論ではなく組織設計の課題として整理し、求心力と遠心力を健全に設計することを支援します。*

## 4. 顧客の悩みの三層構造

> *この三層構造は氷山モデルと直結する。より深い分析はSection 14（SSR理論）とSection 26-3（氷山モデル）を参照。*

### 第一層：目に見えている「現象」

- 採用してもいつの間にか辞めていく
- 社員が悪いわけではない。でも、言われたことはやるが、それ以上は動かない
- 評価や給与の話になると、なんとなく気まずい空気になる
- 研修や面談など手は打ってきた。でも、やるたびに「またか」という雰囲気が漂う

### 第二層：繰り返されることへの「徒労感」

- 自分なりに社員のことを考えてきた。給与も上げた、環境も整えた。なのに、なぜ変わらないのか
- 「うちの会社が好き」と言ってくれる社員がいる一方で、なぜか人が辞めていく
- 「自分の伝え方が悪いのか」と、原因が自分に向かい始めている

### 第三層：言葉にならない「孤独と葛藤」

- 社員には経営者として弱みを見せられない
- 経営者仲間に話すと「社員に問題があるのでは」という方向になる。でも自分はそう思いたくない
- ビジョンはある。想いもある。でも、それが組織に伝わっていないもどかしさを誰にも打ち明けられない

### 問題の本質

*このタイプの経営者が抱える問題の本質は「人間関係の崩壊」ではありません。「ビジョンと現場のあいだをつなぐ言語・仕組みが存在しない」という構造的なギャップです。構造さえ整えば、すでにある人間関係の質がレバレッジとして機能します。*

### A4一枚評価制度®とグロウスリー支援の関係

A4一枚評価制度®は、MillReef 榎本あつし氏のメソッドであり、グロウスリーは養成講座修了・コンテンツ使用許諾を得て活用している。

元メソッドとしてのA4一枚評価制度®は、業績向上と人材育成につながる行動を現場で増やすための評価制度として扱う。公平性・納得性や賃金連動を主目的として説明しない。

グロウスリーの支援では、この考え方をベースに、評価面談・1on1・日常の声かけ・役割期待の整理と接続する。必要に応じて、社労士・人事制度支援として賃金制度や労務面の整合性も別途整理する。

**表現ルール**：A4一枚評価制度®そのものと、グロウスリーが提供する人事制度・賃金制度・対話設計の統合支援を混同しない。

## 10. 100人に1人に刺さる課題意識リスト【完全版】

*使い方：セールスレター・LP・メルマガ・SNS投稿の冒頭で3〜5個を選び「こんな経験はありませんか？」と問いかける。1投稿＝1課題として深掘りする使い方も有効。10カテゴリ×10項目の完全版。*

### 【①採用・定着】
- 採用してもいつの間にか辞めていく。離職理由も「一身上の都合」でよくわからない
- 「うちの会社が好きです」と言って入ってきた社員が3ヶ月で辞めた
- 採用コストを計算するのが怖い
- 「長く働きたい」と言っていた社員が、1年後に転職サイトを見ているのを知ってしまった
- 面接では良い人だと思ったのに、入ってみると全然違う
- 離職率を下げたくて給料を上げたが、それでも辞める人が出る
- 求人票に何を書けばいいかわからなくなってきた
- 辞めていく社員が「一身上の都合」しか言わない
- 採用した人が即戦力になるまで育てたら、その人が辞めた
- 「御社に長くいたいと思えない」という退職理由を聞かされて、何も言い返せなかった

### 【②評価・賃金】
- 「なぜ自分の給料はこの金額なんですか」と聞かれて、うまく答えられなかった
- 頑張っている社員と頑張っていない社員の給料がほとんど変わらない
- 評価制度を作ったことがあるが、結局うまく使えなくて形骸化してしまった
- 賃上げをするとき、誰をどれだけ上げるかの基準がなく、毎回なんとなくで決めている
- 「公平に評価しているつもり」だが、社員にそう思われているかどうかがわからない
- 昇給の面談がいつも気まずい
- 仕事ができる社員ほど「もっと評価されたい」と転職していく
- 同業他社の給料水準を調べると、うちは低いのか高いのかよくわからない
- パートと正社員の仕事の境界線があいまいになってきて、どう処遇すべきか迷っている
- 「評価してもらえていない」という社員の声を間接的に聞いた

### 【③社員の成長・育成】
- 言われたことはやるが、それ以上は動かない
- 若い社員に仕事を教えても、すぐ辞めてしまう
- 管理職にしたのに、プレイヤーのままで全然マネジメントをしてくれない
- 研修に行かせても、翌週には何も変わっていない
- 「成長したい」と言って入ってきた社員が、半年後にはやる気をなくしている
- 社員が失敗を恐れて、新しいことに挑戦しない
- フィードバックをしても「わかりました」とだけ言って、何も変わらない
- 10年いる社員が、新入社員と同じレベルのことをしている
- どうやって部下を育てればいいか、自分自身が誰にも教わったことがない
- 「もっとこうしてほしい」と思っているのに、うまく伝えられていない自分がいる

### 【④組織・チーム】
- 部署間の仲が悪い。原因はわかっているが、どう解決すればいいかわからない
- 古参社員が新しい社員をいびっている気配がある
- 社内の雰囲気が悪いわけではないが、なんとなくギスギスしている感覚がある
- 会議をしても毎回同じ人しか発言しない
- 「うちの会社、変わらないな」と思われていそうで、社員の顔色をうかがってしまう
- 部門をまたいだ協力ができない
- 社長の自分が現場に口を出しすぎているのはわかっている。でも任せると不安
- 社員同士の関係性は悪くない。なのになぜか組織として前に進まない
- 優秀な社員が突然「辞めたい」と言ってくる
- 幹部候補だと思っていた社員が、急に主体性を失った

### 【⑤経営者自身の孤独・葛藤】
- 人事・組織の悩みを誰にも相談できない
- 「社長なんだから強くなければ」と思っているが、本当はもう限界に近い
- 社員には絶対見せられないが、「もうやめたい」と思う夜がある
- 経営者仲間に相談すると「社員のせいだ」という話になる。でも自分はそう思いたくない
- 「自分のやり方が間違っているのかも」と思い始めると、何も決断できなくなる
- 社員のことを考えて動いているのに、なぜ伝わらないのかが本当にわからない
- 家族に会社の話をすると心配されるだけなので、もう話さなくなった
- 「経営者は孤独」という言葉の意味が最近やっとわかってきた
- 誰かに「それで合ってますよ」と言ってほしいだけなのかもしれない
- 自分が倒れたら会社はどうなるのか、と考えると怖くて眠れない夜がある

### 【⑥特定の社員・人間関係】
- 辞めてほしい社員がいる。でも面と向かって言えないし、言い方もわからない
- 古参社員が変化を嫌がって、改革のブレーキになっている
- 特定の社員が他の社員の雰囲気を悪くしている。でも証拠がない
- 社員に好かれようとしすぎて、言いたいことが言えなくなっている
- 問題社員だと思っていたが、実は自分の関わり方に問題があったのかもしれない
- できる社員に依存しすぎていて、その人が辞めたら会社が回らない
- 「この社員とは相性が合わない」と感じているが、それを認めることへの罪悪感がある
- 社員を信頼して任せると裏切られる、という経験を何度もしてきた
- 社員の前では笑顔でいるが、実は特定の社員に対してかなりの不信感を持っている
- 「なぜあの人が辞めないのか」と思う社員がいる

### 【⑦制度・仕組み】
- 就業規則があるが、最後に見たのはいつだったか思い出せない
- 人事制度を作りたいと思って調べたが、複雑すぎて何から始めればいいかわからなかった
- 目標管理制度を導入したが、形だけになって機能していない
- 1on1をやり始めたが、何を話せばいいかわからず気まずい時間になっている
- 「うちの規模には人事制度は早い」と言われたが、本当にそうなのか疑問がある
- 社員に期待していることを言語化しようとすると、うまく言葉にできない
- ルールを作ると守らない社員が出る
- 会社の理念を作ったが、額縁に飾っているだけで誰も意識していない
- 「仕組み化しなければ」とわかっているが、日々の業務に追われて手が回らない
- 組織図を作ったが、実際の仕事の流れと全然違う

### 【⑧将来・成長への不安】
- このままの組織では3年後・5年後が見えない
- 社員は増えているのに、売上が比例して伸びない
- 後継者を育てたいが、誰をどう育てればいいかの基準がない
- 「人が育つ会社」にしたいという夢がある。でも具体的に何をすればいいかわからない
- 社員を増やすほど管理が大変になる。スケールするほど疲弊するのはおかしい
- 社員に「この会社で働いてよかった」と思ってほしい。でも今は自信を持って言えない
- 業績は悪くない。でも組織の雰囲気が好きになれない
- 自分がいなくても会社が回る状態にしたいが、何年経っても実現できていない
- 「良い会社だね」と言われる会社にしたい。でもその定義が自分の中で定まっていない
- このまま社員と一緒に成長できる会社になれるのか、最近自信がなくなってきた

### 【⑨経営と人事の接続】
- 経営計画は作るが、それが社員の行動に結びついていない
- 「人件費が高い」と感じているが、何を基準に判断すればいいかわからない
- 粗利や生産性の視点で人員配置を考えたことがない
- 組織の問題と業績の問題が連動していると頭ではわかっているが、具体的にどう動くか
- 採用基準が明文化されていないため、毎回違う判断をしている
- 「人への投資」がどれだけ回収できるか、計算したことがない
- 社員一人ひとりの強みを活かせているか、客観的に見えていない
- 経営理念と現場の行動がつながっていない
- 財務数字は見るが、人事数字（定着率・生産性・エンゲージメント）を見たことがない
- 「良い組織をつくれば業績が上がる」と信じているが、その根拠を言語化できていない

### 【⑩変化・改革への抵抗】
- 変えたいと思って動くたびに、社員の反応が薄くて諦めてしまう
- 「また社長の思いつきが始まった」と思われていそうで、新しいことを言い出せない
- 変化を嫌う社員が多い
- 改革しようとするたびに古参社員が「前はこうだった」と言い出す
- 外部のコンサルに入ってもらったが、社員の反発が強くて定着しなかった経験がある
- 「変えよう」と言うと「また変えるんですか」と言われる
- 自分が変わろうとしているのに、周りがついてこない孤独感がある
- 小さな改善を積み重ねてきたが、根本的なところが何も変わっていない気がする
- 「うちの会社の文化だから」という言葉が改革の壁になっている
- 変えたいが、変えることで今うまくいっていることが崩れるのが怖い

## 12. 原体験・創業ストーリー

### フル版（講演・プロフィール・セールスレター用）

社会人として働き始めた頃、月約150時間の残業が1年半続き、過労で潰瘍性大腸炎を発症。仕事自体はお客様に可愛がられ楽しかったものの、「健康でなければ働き続けることすらできない」という現実を痛感しました。

転職した会計事務所では15年間、多くの中小企業の経営者・社員の労務トラブルに関わる中で、「経営者も社員も、それぞれ本気で会社のことを考えている」という事実に気づきます。それなのに、価値観や立場の違いから善意がかみ合わず、すれ違いが生まれ続けている現実がありました。

そこで見えてきたのが、「仕事がたのしくなくなる本質は、やらされ感と自己にとっての意味の喪失にある」という洞察です。チームビルディングと粗利を軸とした管理会計を組み合わせ、経営者も社員もお客様も結果として満足度が上がるゴール設定を支援したい——その思いでグロウスリーを設立しました。

## 14. SSR理論（人材力・組織力・関係力）

*使い方：グロウスリーの3サービスがなぜ一体で機能するかの構造的根拠。「なぜワンストップで提供するのか」への回答として使う。*
*帰属：チームビルディングSSR理論（株式会社コーチングファームジャパン）の考え方を支援の土台として活用しています。*

### SSR理論とは

SSR理論は、組織づくりのための3つの要素（人材力・組織力・関係力）を提案する理論。この3要素を個人レベル→チームレベル→会社レベルへとスパイラル状にレベルアップさせることで、人と業績が同時に成長し成果につながる。

### 3要素の定義とサービスとの対応

| 要素 | 定義・問い | グロウスリーのサービス | マーケティング活用 |
|---|---|---|---|
| **人材力（Strengths）** | 個人の強み・弱み・思考行動の特徴を見る視点。能力不足と決めつける前に、その人の力が発揮される役割や環境になっているかを見立てる。 | 思考スタイル診断（効き脳）／研修 | 「気づきをつくる研修」の根拠として使う |
| **組織力（Structure）** | 目的・目標・役割・ルール・評価・仕組みを見る視点。人が迷わず動けるように、何を大切にし何を期待しているのかを言葉にする。 | A4一枚評価制度®をベースにした人事制度設計 | 「組織力を整える中核支援」として制度設計を位置づける |
| **関係力（Relation）** | 対話・承認・フィードバック・話し合いを見る視点。必要なことを伝え合える関係になっているかを見立てる。 | 1on1設計／チームビルディング研修／伴走支援 | 「現場に根づく伴走支援」の理論的根拠として使う |

### SSRサイクル（好循環の構造）

人材力（強みに着目）→組織力（強みを活かす目標設定・制度設計）→関係力（目標達成のためのコミュニケーション）→人材力（さらに深い強みの発見）という好循環サイクル。

### 組織成長の5階層

| 階層 | 単位 | 問い・状態 |
|---|---|---|
| 1階層 | 社員個人 | 社員が人の多様性・会社ビジョン・リーダーについてどう捉えているか |
| 2階層 | 個人と所属する部署 | 会社や部署のビジョンが明確で、仕事を通じた自己成長を感じているか |

## 17-2. 1on1の3つの支援

*使い方：1on1を単なる進捗確認や本音を聞き出す場として扱わず、短期成果と中長期成長のバランスを整える対話の仕組みとして説明する。*

**提案時の表現例**：1on1は、進捗確認だけの時間ではありません。業務を前に進め、経験を成長につなげ、働き続けられる状態を整えるための対話の時間です。

## 26. MQ会計の扱い

グロウスリーは、西順一郎先生（西研究所）より「西順一郎先生のMQ会計の考え方をもとに」とクレジットを表記することを条件に、MQ会計の概念・フレームワークをポッドキャスト・セールスレター・提案資料等で自由に使用する許諾を受けています。

## 26-2. A4一枚評価制度®・サポートシート運用の使い方

A4一枚評価制度®を提案・発信で扱う際は、A4一枚の評価シートを作ることだけに閉じない。評価項目を現場で見える行動に分解し、サポートシート／マネジメントシートで期中に思い出し、評価面談や1on1で次の行動につなげる運用まで含めて説明する。

**評価項目の3カテゴリ**：
- 成果・達成評価：短期的な業績や目標達成につながるもの
- 役割・スキル評価：中長期的な成長や、将来担ってほしい役割に向けたもの
- 姿勢・態度評価：日常の行動姿勢や、組織で大切にしたい行動を見るもの

「主体性」「協調性」などの抽象的な評価項目は、その職場で見える具体的な行動に分解する。5つ程度の着眼点にすることで、期中チェックや最終評価で確認しやすくなる。

サポートシート／マネジメントシートは評価期間中に上司と部下がやり取りするための運用ツール。期中運用は30分面談のように重くしすぎず、短く、こまめに、記録が残る形で行う。「ちょっとをちょくちょく」行動を確認し、承認し、必要に応じて軌道修正する。

ToBe（到達したい状態）とToDo（そのための具体行動）を分ける。提案資料では「どうなっていたいか」と「そのために何をするか」に翻訳して使う。

**提案時の表現例**：A4一枚評価制度®は、評価を複雑にするものではありません。業績向上と人材育成につながる行動を、期中のサポートシートや面談でこまめに確認し、現場に根づかせる仕組みです。

## 26-3. 組織開発・チームビルディング補足

### 氷山モデル（4層）
できごと（現象）→時系列パターン→構造（仕組み）→メンタルモデル（前提）の順に見る。目に見える問題をすぐ解決しようとせず、繰り返されるパターン、それを生む構造、背景にある思い込みや前提を見る。

### 効き脳・思考スタイル診断
人を分類するためではなく、「人と人は違う」という前提を共有するために使う。相手の行動を性格や悪意で決めつける前に、自分とは見ているものや判断基準が違うのかもしれないと考える入口にする。

### 経験学習サイクル
育成では、経験させるだけでは不十分。経験したことを振り返り、学びを言葉にし、次の行動に活かすことで成長につながる。1on1・振り返り・評価面談では、「何が起きたか」「自分はどう判断したか」「何を学んだか」「次にどう動くか」を確認する。

### MVVの扱い
VisionはMissionは何のためにそこへ向かうのか。Valueは道中で何を大切にし、何を判断基準にするのか。MVVはきれいな言葉を掲げるためではなく、迷ったときに戻る定位置をつくるものとして扱う。

### 役割と能力発揮
能力があっても、役割が曖昧なままだと力は発揮されにくい。役割の明確化は、個人の能力を引き出すための組織力であり、関係力を機能させる土台でもある。

### 人とことを分ける
対立・不満・評価・賃金の話では、人とことを分ける。人物評価や感情論に寄せすぎず、まず事実・状況・行動に戻って整理する。

### 谷をくぐる／チーム成長
チームが変わるときは、最初から右肩上がりに成果が出るとは限らない。言いづらいことを言葉にする時期や、混乱する時期がある。ブログや研修では、これを悪化ではなく、新しい秩序が生まれる前の必要な過程として扱う。

## 26-4. 行動経済学・行動設計の前提

人は、合理的に正しいとわかっていることをいつも選べるわけではない。目先の緊急度、現状維持、損失や失敗への恐れ、習慣、忙しさ、周囲の空気に影響される。

だからこそ、研修・評価制度・1on1・サポートシートは、人の意志を責めるためではなく、望ましい行動を思い出し、試し、振り返り、続けやすくするための仕組みとして設計する。

**提案時の表現例**：人は、正しいとわかっていても、忙しさや習慣に流されます。だから、思い出せる場や振り返れる仕組みをつくることが大切です。育成や評価を本人の意志の強さに任せず、行動しやすい環境を整える。それが、グロウスリーが考える仕組みづくりです。

## 27. 時代認識と課題設定者としての位置づけ

*使い方：ブログ・提案書・HPコピーを書く際、「役に立つ情報提供」ではなく「経営者が自社の問いに気づく手助け」という視点でコンテンツを設計する。AIへの指示文にもこの軸を明示して使用すること。*

### 時代の構造変化

社会全体の構造的問題は希少化しつつある。AIを含む解決力の向上により、「問題があれば解く」という時代から「何を問題と設定するか」に価値が移っている。一方、現場には「できごと」レベルの問題は豊富に存在する。しかしそれをできごととして解決しても、モグラたたきになることが多い。

マーケティングはマスから個へ移行している。個人の想いがビジネスの種になり、機能ではなく意味を売る時代になっている。

### グロウスリーの役割：問題解決者から課題設定者へ

グロウスリーの役割は「問題解決者」ではなく「課題設定者」である。目に見えるできごとの下にある構造を見立て、経営者が本当に解きたい問いを一緒に言語化する。これが「役に立つ」から「意味がある」へのシフトであり、AIが解決力を持つ時代における人間のコンサルタントの差別化軸になる。

### AIとの役割分担

AIは解決力を持つが、課題設定はできない。「何が本当の問いか」を経営者と一緒に見立てることが、これからの中小企業支援における人間の専門家の核心的な役割である。

## 28. 経営哲学的背景（二宮尊徳・渋沢栄一）

*使い方：経営者向けの提案書・ブログ・ポッドキャストで「なぜ短期的成果より組織の土台を整えることを優先するのか」への答えとして使う。*

### 核心となる言葉

> **「道徳なき経済は犯罪であり、経済なき道徳は寝言である」**（二宮尊徳）

渋沢栄一の「論語と算盤」も同じ思想の系譜にある。「利益を生み出し、納得して分かち合える組織をつくる」というグロウスリーの一言ポジションは、この思想の現代的実装である。

- 「利益を生み出す」＝経済なき道徳は寝言。想いだけでは組織は動かない
- 「納得して分かち合える」＝道徳なき経済は犯罪。利益だけ追えば組織は壊れる

### 4つの実践軸

**①短期成果と中長期成果の両立**：研修・制度・伴走は即効性より組織の土台を育てることを優先する。モグラたたきで終わる短期施策より、構造を変える中長期支援に価値がある。

**②法令遵守を超えたコンプライアンス**：「なぜこの評価なのか」「なぜこの賃金なのか」を説明できることが、持続可能な組織の条件である。

**③世間常識・認識変化への対応**：時代の変化に合わせて制度や対話の形をアップデートし続けることがビジネスの持続可能性につながる。

**④意味と経済の同時設計**：社員の成長実感（意味）と、会社の利益（経済）は対立しない。両者を同時に設計できるのが、グロウスリーのA4一枚評価制度®と伴走支援の核心的な価値である。

## 37. 経営数値と人事制度の連動モデル（外部視点と賃金原資の設計）

*使い方：経営者向け提案書・初回相談・ブログで「なぜ人事制度と経営数値をセットで考えるのか」を説明する際に使う。会計事務所15年の経験がこの視点の信頼根拠になる。*

### 前提認識：インフレと賃金の現実

2021年以降の5年間で物価は13%以上上昇している。賃金水準を据え置くことは、実質的な賃下げと同じ意味を持つ。賃金原資を確保しながら経営状況を悪化させず、顧客への貢献を高めていくことは、今や労使共通の課題である。

### 経営数値と人事の一気通貫モデル

```
外部環境（物価上昇・市場変化・顧客ニーズ）
　↓
経営数値の把握（粗利率・売上総利益・労働分配率・人時生産性）
　↓
賃金原資の設計（分配できる利益をどう生み出すか）
　↓
人事制度（評価・賃金・役割定義：納得感のある分配基準の言語化）
　↓
組織づくり（研修・1on1・伴走：行動変容・成長実感・顧客貢献の習慣化）
　↓
顧客への貢献（粗利の創出・リピート・紹介）
　↓（循環）
賃金原資の拡大→再分配→さらなる貢献
```

この全段を見立てられる支援者は少ない。研修会社は下2段、一般的な社労士は人事制度周辺のみ。会計事務所15年×社労士×組織開発というバックグラウンドがこの視点の根拠になる。

### 重要な経営指標と人事制度の接続

| 経営指標 | 意味 | 人事制度・組織づくりとの接続 |
|---|---|---|
| 売上総利益（粗利） | 売上－変動費。分配できる利益の源泉。 | 粗利が増えなければ賃上げの原資は生まれない。粗利を増やす行動を評価制度に組み込む。 |
| 粗利率 | 粗利÷売上。業務の付加価値の高さを示す。 | 粗利率が低い会社は「安売り」か「非効率」のどちらか。人材の強みを活かした仕事の再設計が必要。 |
| 労働分配率 | 人件費÷粗利。粗利のうち人件費に回る割合。 | 適正な労働分配率を設計することで、賃上げと経営安定の両立ラインが見える。 |
| 人時生産性 | 粗利÷総労働時間（MQ÷総労働時間）。一人1時間あたりの付加価値。 | 人時生産性を上げることが賃上げの根拠になる。研修・制度・1on1はすべてここに収束する。 |

### 賃金原資確保の実践的な考え方

- 人時生産性が上がれば粗利が増え、労働分配率を維持しながら賃金を上げられる
- 成長実感・役割明確化・評価の納得感が整うと、離職コストが減り、実質的な賃金原資が増える
- 顧客への貢献が増えると売上・粗利が上がり、分配の原資が自然に拡大する

この循環を設計することがグロウスリーの支援の核心であり、「道徳なき経済は犯罪であり、経済なき道徳は寝言である」（Section 28）という哲学の実装でもある。

*AIへの注意：労働分配率・人時生産性・粗利率の数値は業種・規模によって大きく異なる。具体的な数値を提示する際は「目安」として扱い、断定しないこと。*

### ブログの戦略的位置づけ（v12追加）

グロウスリーのブログは、単なる情報提供やSEO記事ではなく、読者が自社の組織課題を別の角度から捉え直すための入口として位置づける。

**ブログの役割は、読者にすぐ答えを渡すことではなく、問いを変えることにある。**

問いの転換の例：
- 社員が悪いのか、動きにくい場になっているのか。
- 管理職の力量不足なのか、役割と相談先が曖昧なのか。
- 信頼が足りないのか、信頼を支える情報と判断基準が足りないのか。
- 組織が変わっていないのか、変化のサインを受け取れていないのか。

この「問いの転換」を通じて、読者が自社の問題を一人で抱えるのではなく、グロウスリーに相談する入口をつくる。

### ブログカテゴリーごとの役割定義（v12追加）

**経営者の悩みをほどく**
経営者の自責・苛立ち・不安を、性格や器の問題ではなく、情報・判断基準・見方・対話の設計として捉え直すカテゴリー。読者に「自分が悪いのではなく、整理できていない構造があるのかもしれない」と感じてもらう。

**現場に根づく実践**
制度や方針を、現場の日常行動に落とすカテゴリー。相談が早くなる、会議で問いが出る、経過を語る言葉が増えるなど、組織の変化を日常の場面として描く。

**人事制度・評価・賃金**
制度の正しさだけでなく、社員がその制度をどう受け取り、どんな行動につながるかまで扱うカテゴリー。評価制度・賃金制度・目標設定を、社員の認知や行動の変化と接続して考える。

**組織づくりの考え方**
グロウスリーの思想や見立てを伝えるカテゴリー。理想論だけで終わらせず、日常の対話・役割・承認・判断・見方の更新に戻して描く。「月曜日が待ち遠しい組織」は、完成された理想像ではなく、日々の関わりの積み重ねによって少しずつ近づいていく状態として扱う。

### 第5シーズンの完了と位置づけ（v12追加）

第5シーズンは、制度・評価・1on1・会議・承認・判断・経営者自身の見方を扱いながら、組織づくりを「完成させるもの」ではなく「更新し続けるもの」として捉え直すシリーズとして完了。最終的にはグロウスリーのビジョンである「月曜日が待ち遠しい組織」に接続する。月曜日が待ち遠しい組織は、理想論として語るのではなく、日々の対話・相談・承認・判断・役割の言葉の積み重ねによって少しずつ近づいていくものとして描く。

**シーズンテーマ**：人は、正しさだけでは動かない。だから、仕組みと対話を設計する

**第1〜4シーズンとの位置づけ**：第1〜4シーズンが「何を整えるか」の実践論。第5シーズンは「なぜ人はそう動くのか」の人間理解論。

各話タイトルと設計は別添の第5シーズン確定版を参照。

## 39. ステップメール設計 v3（14日・全7回版・認知変化型）

### 設計概要

| 項目 | 内容 |
|---|---|
| **通数** | 全7回 |
| **期間** | 14日（登録直後〜14日後） |
| **登録フォーム** | https://docs.google.com/forms/d/1Hx2X7opKKJJv7rM_USoi8CfVVuJSCXRR1pDLNg_sx-g/viewform |
| **管理スプレッドシート** | GAS連携・subscribersシートで一元管理 |
| **目的** | 経営者の問いの転換→信頼形成→初回相談申し込み |
| **設計方針** | 施策別の横並びではなく、経営者の認知が変わる順番で設計する |

### 位置づけ

ステップメールは、サービス説明のための連続メールではなく、経営者が自社の組織課題を別の角度から捉え直すための導線として設計する。

旧版は「研修」「採用」「評価」「管理職」など施策別に課題を並べていたが、新版では「経営者の認知が変わる流れ」を重視する。

目的は、読者に「自社の問題は人の問題だけではなく、場・言葉・制度の使われ方・変化の受け取り方の問題かもしれない」と感じてもらうことにある。

最終的には、「相談は整理できてからではなく、整理できないから始める」という感覚につなげる。

### 全体構成

| 通 | 件名 | 役割 |
|---|---|---|
| 1 | 「何度伝えても変わらない」は、人の問題ではないかもしれない | 問いの転換・入口宣言 |
| 2 | 動かない社員の前に、動きにくい場がある | 人の問題から場の問題への見立て直し |
| 3 | 社長の言葉は、なぜ現場で別の意味になるのか | 伝達と意味づけの問題を扱う |
| 4 | いい制度なのに使われない会社で起きていること | 制度の中身ではなく、使う場面の設計へ視点を移す |
| 5 | 社員が動き始めた会社に増えるサイン | 変化のサイン・自己診断 |
| 6 | 「いつになったら整うのか」と焦っている経営者へ | 組織づくりを完成ではなく更新として捉え直す |
| 7 | 相談は、整理できてからでなくていい | 相談導線・心理的ハードルを下げる |

### 配信スケジュール（delay_days設定）

現行GASの `delay_days` 設定を踏襲する。

| 通 | delay_days | 配信タイミング | 件名 |
|---|---:|---|---|
| 1 | 0 | 登録直後 | 「何度伝えても変わらない」は、人の問題ではないかもしれない |
| 2 | 1 | 1日後 | 動かない社員の前に、動きにくい場がある |
| 3 | 3 | 3日後 | 社長の言葉は、なぜ現場で別の意味になるのか |
| 4 | 5 | 5日後 | いい制度なのに使われない会社で起きていること |
| 5 | 7 | 7日後 | 社員が動き始めた会社に増えるサイン |
| 6 | 10 | 10日後 | 「いつになったら整うのか」と焦っている経営者へ |
| 7 | 14 | 14日後 | 相談は、整理できてからでなくていい |

### 設計思想

**旧版からの変更点：**
- 「研修」「採用」「評価」「管理職」など施策別に並べる構成から、経営者の見方が変わる順番へ変更する
- 読者にノウハウを一方的に渡すのではなく、「自社では何が起きているのか」を考え直す体験にする
- 各通を「1通1メッセージ・1通1問い」に絞る
- 最終通までは売り込みを強めず、7通目で初回相談への導線を明確にする

**各通の流れ：**
- 1通目・2通目では、「社員が変わらない」という見方を、「動きにくい場があるのではないか」という問いに変える
- 3通目では、社長の言葉が現場で別の意味になる理由を、伝え方の問題ではなく、前提と意味づけの違いとして扱う
- 4通目では、制度やシートが使われない理由を、内容の良し悪しではなく、使い始める場面が設計されているかという視点に置き換える
- 5通目では、変化を数字だけで確認するのではなく、相談・問い・経過の言葉といった日常のサインから受け取る視点を伝える
- 6通目では、組織づくりを完成させるものではなく、更新し続けるものとして捉え直す
- 7通目では、問題が整理できていない段階こそ相談の入口であると伝え、初回相談へ自然に誘導する

### CTA方針

1〜6通目では、強い売り込みはしない。各通の最後に次回導線を置き、読者の中に問いを残す。

7通目で初回相談への導線を明確にする。

**7通目の相談ハードル解除フレーズ（必須）：**
```

答えを持っていく場ではありません。

一緒に状況を整理する時間です。

仕組みの話として
まとまっていなくても大丈夫です。

気になっていることをそのまま
話すところから始められます。
```

**CTA表記：**
```

https://grow3.jp/contact2/
```

## 41. 理論的背景（v12新設）

*使い方：ブログ執筆・研修設計・提案書作成の際に、どの理論をどう扱うかを判断するための内部リファレンス。理論名を公開文書の前面に出さないためのガイドとして機能する。*

### 理論の3層構造

#### 底流（思想の土台として全体に流す・本文では用語を出さない）

| 理論 | 本文への翻訳例 |
|---|---|
| 社会構成主義 | 「日々の会話や会議、評価面談や1on1の中で、組織の空気は少しずつつくられ続けています」 |
| 成人発達理論 | 「幹部が伝える段階から翻訳する段階へ移る」 |
| 選択心理学（底流として） | 「人が仕事の中で求めているもの」として現場の言葉で表現する |

#### 表層（記事の骨格として正面から扱う・用語は本文末尾に1つまで）

| 理論・概念 | 使い方・注意 |
|---|---|
| 現状維持バイアス | 「変化のコストを無意識に大きく感じる」として現場の場面から入る |
| ナッジ・デフォルト効果 | 「動きやすい初期設定の設計」「何もしなければ使われないという前提で設計する」 |
| 損失回避 | 「失うことへの不安が、得ることへの期待より大きく見える」 |
| 確証バイアス | 「一度持った見方に合う情報を拾いやすくなる」 |
| 承認3分類（存在・経過・結果） | そのまま使用可。初出時は「存在・経過・結果という3種類の承認」と説明を入れる |
| 選択心理学5欲求 | 「人が仕事の中で求めている5つのもの」として文脈に合わせて表現する |
| 行動特性3軸 | 「自己表現・自己主張・柔軟性という動き方の初期値」。shindan.htmlとの接続あり |
| 社会構成主義（91話のみ正面から） | 91話に限り用語として本文末尾に登場させてよい |

#### 実践フレーム（診断・研修導線との接続）

| フレーム | 接続先 |
|---|---|
| 行動特性3軸 | shindan.html（思考スタイル診断）との接続。診断ツールへの誘導CTA設計に活用 |
| ヘルマンモデル | 研修設計・チームビルディング・1on1設計の理論根拠として使用 |

### 使用原則

- 理論名は本文末尾に小さく登場させる。本文は現場の言葉で書く
- 1記事に理論名は1つまで
- 底流理論（社会構成主義・成人発達理論・選択心理学）は翻訳して使い、用語として出さない
- 表層理論でも冒頭に出さない。現場の場面→理論の順を守る
- 理論説明は原則2〜3文以内に抑える
- 理論は読者を説得するためではなく、日常の違和感に名前を与えるために使う
