**本ドキュメントおよび参照するfixture内の数値・企業名・決算データはすべて架空である。
実在の企業のものではない。** `docs/financial-analysis_指標定義書_v0.2.md` の全指標に

## 1. 本体：架空製造業「A社」3期分

### 1-1. 設計思想

- 単一製品に単純化（P・V・Q・M・F・Gが直接対応する形にし、MQ会計の検算を
  容易にする）。
- B/S科目は最小構成：現金預金・売掛金・棚卸資産・有形固定資産(純額)・
  買掛金・借入金・資本金・利益剰余金の8科目のみ。
- 現金預金は「他の科目を先に決め、貸借一致となるよう期首時点で逆算し、
  以降は複式簿記の恒等式（期末現金＝期首現金＋営業CF＋投資CF＋財務CF）で
  機械的に前進させる」方式で構築。これにより各期末で
  `資産合計－（負債＋純資産）＝0`（丸め誤差1e-9オーダー）が自動的に成立する
  ことをコードで検証済み（`balance_check.diff` を参照）。
- 借入金は期首500,000千円（過去の設備投資等で借入依存度が高い前提）、
  利益剰余金は期首▲10,000千円（過去の累積損失が薄く残るが直近3期は
  黒字化途上、という設定）。
- 維持投資（capex）＝減価償却費と一致させ、有形固定資産は3期を通じて
  純額200,000千円のまま横ばい（§18「維持投資≒減価償却費」前提と整合）。
- 年間元本返済額は3期とも20,000千円で固定（返済スケジュール自体は変えず、
  「稼ぐ力の変化」だけでCF自走性の可否が変わることを見せるため）。

### 1-2. 3期のストーリー

| 期 | 位置づけ | 売上高 | 当期純利益（税務損益） | CF自走性 |
|---|---|---|---|---|
| 第1期 | 安定期 | 800,000千円（Q=10,000個×P=80千円） | 26,000千円（黒字） | **1.30（標準）** |
| 第2期 | 急成長期・必須シナリオ | 1,000,000千円（Q=12,500個・+25%） | 67,860千円（黒字・増益） | **▲0.94（要改善）** |
| 第3期 | 回復・安定化期 | 1,050,000千円（Q=13,125個・+5%） | 80,015千円（黒字・増益） | **6.17（余裕）** |

**必須シナリオの成立根拠（第2期）**：急成長により販売数量が+25%増加する一方、
回収サイトが悪化（売掛金滞留日数45→55日）、在庫も積み増し（棚卸資産滞留
日数60→75日）。この結果、当期純利益は前期の26,000千円から67,860千円へと
大幅増益（税務上は明確な黒字拡大）でありながら、推計営業CF（P/L＋2期分
B/Sから間接法準拠で推計。本fixtureにはCF計算書が無いため`ocf_source=
estimated`）は売掛金・棚卸資産の増加により1,284.7千円まで圧縮され、
FCF（推計営業CF−維持投資20,000千円。維持投資は実額`capex`を使用し
`capex_source=maintenance_actual`）は▲18,715.3千円のマイナスとなる。年間返済元金
20,000千円に対しCF自走性＝▲0.94倍で「要改善（自走していない。最優先論点）」
と判定される。**税務損益（黒字・増益）よりもCF自走性（第0指標）が優先して
警告を出す**、という§18の思想どおりの構造になっている。

> **重要な注記（2026-07-12追加）**：第2期は運転資本変動（売掛金・棚卸資産の
> 滞留日数の悪化）が大きいため、**簡易営業CFでの正式判定は行えない**
> （§1・§18の2026-07-12訂正：簡易営業CFは速報・一次スクリーニング専用）。
> 本fixtureは前期末B/S（第1期末）を保有しているため推計営業CFが算出でき、
> 正式判定（CF自走性＝▲0.94・要改善）が可能になっている。もし前期末B/Sも
> CF計算書も無く簡易営業CFしか算出できない入力だった場合、
> `financial_calc.py`は判定値を出さず「運転資本変動を確認できないため
> 正式判定不可」の警告を返す設計とする（`tests/fixtures/
> financial_analysis_boundary_cases.json`の`ocf_source_safety_cases`を参照）。

### 1-3. 第2期についての重要な補足（簡易指標では危険を検知できない）

同じ第2期について、§2の「年間元本返済負担率」（簡易営業CFを分母とする
簡便指標。速報・一次スクリーニング専用）を計算すると **0.295倍**
（返済原資CF＝簡易営業CF87,860−維持投資20,000＝67,860千円から算出。
簡易営業CFは経常利益＋減価償却費−法人税等ベースで運転資本の悪化を
反映しない）となり、一見「問題なし」に見えてしまう。これは
CF自走性（推計営業CFベース・運転資本の悪化を反映する）が簡易指標より
厳格に、かつ早期に危険信号を出すことを示す実例であり、§18が
「税務損益ではなくCF自走性を第0指標に置く」設計にした理由を、
このfixtureの中で定量的に再現している。**financial_calc.py実装時は
この2指標（簡易版の年間元本返済負担率／CF自走性）が別々に計算され、
混同されないことをテストで確認すること。**

### 1-4. 余剰現預金ゾーンとsurplus_cashの見かけ上の矛盾（設計上の注意点）

第1期・第3期は「現預金÷月商」が3倍を超え§17の「安全〜戦略判断ゾーン」に
該当するが、同時に「余剰現預金」の値自体は3期ともマイナス（正常運転資金＋
月商3か月分の必要バッファが現預金を上回っている）。これは矛盾ではなく、
**ゾーン判定（月商倍率のみで見る簡便な一覧性指標）と余剰現預金の金額計算
（運転資金所要額まで差し引く精緻な指標）は別の情報を示している**ため。
実装時、この2つを同じ意味の指標として混同しないこと（テストケースとして
両方を明示的に検証する）。

### 1-5. B/S整合性の検証結果

3期すべてで `balance_check.diff`（資産合計－負債純資産合計）が0または
浮動小数点丸め誤差（最大1.16×10⁻¹⁰千円）に収まることをコードで確認済み。
純資産の期間推移（期末利益剰余金＝期首利益剰余金＋当期純利益、配当なし）、
現預金の期間推移（期末現金＝期首現金＋営業CF＋投資CF＋財務CF）も
fixture内の`opening_bs`／`ending_bs`／`cf`を突き合わせることで検証可能。

### 1-7. 手計算の式展開（代表例・第2期）

以下は第2期の主要指標について、fixtureの入力値から結果までの式展開を
示す（`build_financial_fixture.py`のcompute関数群と同一の計算）。

```
売上高 = Q(12,500個) × P(80千円/個) = 1,000,000千円
変動費 = Q(12,500個) × V(48千円/個) = 600,000千円
MQ = 売上高 − 変動費 = 400,000千円

固定費合計 = 132,000+40,000+16,000+58,000+10,000+4,000+5,500+1,000+19,500
           = 286,000千円
営業利益(G) = MQ − F = 400,000 − 286,000 = 114,000千円

支払利息 = 期首借入金(第1期末残高480,000千円) × 2% = 9,600千円
経常利益 = 114,000 − 9,600 = 104,400千円
法人税等 = 104,400 × 35% = 36,540千円
当期純利益 = 104,400 − 36,540 = 67,860千円 … 黒字（増益）

売掛金期末 = 1,000,000 ÷ 365 × 55日 = 150,684.93千円
棚卸資産期末 = 600,000 ÷ 365 × 75日 = 123,287.67千円
買掛金期末 = 600,000 ÷ 365 × 30日 = 49,315.07千円

Δ売掛金 = 150,684.93 − 98,630.00(第1期末) = 52,054.93千円（増加＝資金流出）
Δ棚卸資産 = 123,287.67 − 78,904.00(第1期末) = 44,383.67千円（増加＝資金流出）
Δ買掛金 = 49,315.07 − 39,452.00(第1期末) = 9,863.07千円（増加＝資金流入）

推計営業CF（ocf_source=estimated。本fixtureはCF計算書非保有のため
P/L＋2期分B/Sから間接法準拠で推計） = 当期純利益 + 減価償却費 − Δ売掛金
           − Δ棚卸資産 + Δ買掛金
           = 67,860 + 20,000 − 52,054.93 − 44,383.67 + 9,863.07
           = 1,284.66千円（純利益67,860千円に対し極端に圧縮）

FCF = 推計営業CF − 維持投資(実額capex=20,000。capex_source=maintenance_actual。
      本fixtureは実額を保有しているため減価償却費への代理は行わない）
    = 1,284.66 − 20,000 = ▲18,715.34千円

CF自走性 = FCF ÷ 年間返済元金(20,000) = ▲18,715.34 ÷ 20,000 = ▲0.94倍
        → ゾーン：要改善（自走していない。最優先論点）
```

（数値は`tests/fixtures/financial_analysis_case_manufacturing_3fy.json`の
第2期`cf`・`indicators`キーと完全一致。厳密値はJSON参照。）

## 3. financial_calc.py実装時の参照方法（想定）

1. `tests/fixtures/financial_analysis_case_manufacturing_3fy.json` の各期
   `pl`・`cf`・`opening_bs`・`ending_bs` を入力として`financial_calc.py`の
   各関数に渡し、戻り値が同ファイルの`indicators`と一致することを検証する。
2. `tests/fixtures/financial_analysis_boundary_cases.json` の
   `cf_self_sufficiency_boundary_cases`・`debt_payback_ebitda_boundary_cases`・
   `cash_months_boundary_cases`（計12ケース）の`inputs`を関数に渡し、
   `expected`と一致すること（特に`zone`の文言・境界の＜／≦の向き）を検証
