# Grow3 ブログ概念対応表

> 目的：ブログ・ステップメール執筆時の概念のブレを防ぐ。
> 使い方：この言葉を記事で使うとき、対応するframeworksのエントリを意識して書く。本文に `[→F:]` は入れない。
> 参照元：`grow3-frameworks.md`（理論の地図）、`グロウスリー_ブログ執筆コンテキスト_v3.md`（文体・禁止語）
> 原則：本文では理論名・フレーム名を前面に出さない。必要な場合のみ、記事末尾や補足で最小限に触れる。本文は「経営者の言葉」で書く。

## 概念対応表

| ブログで使う言葉・場面 | 対応するframeworksエントリ | ブレやすい点・注意 | 外向きの言い換え例 |
|---|---|---|---|
| 組織の目詰まり・同じ問題が繰り返される | システム思考／構造原型（システム・アーキタイプ） | 「人の問題」に寄せない。構造・仕組みの問題として描写する | 「同じ部署で同じ問題が繰り返される理由は、人ではなく仕組みにある」 |
| 人ではなく構造を見る | システム思考／認知の非対称性が構造的問題を引き起こす | 犯人探しや責任追及に滑らない。「誰のせいか」でなく「どんな構造か」へ | 「誰が悪いかより、どんな構造がこれを生んでいるかを見る」 |
| 仕組みと対話 | SSR理論（人材力・組織力・関係力）／社会構成主義（対話が現実をつくる） | 「仕組みだけ整えれば動く」「対話だけすれば動く」どちらにも寄せない。3要素は連動 | 「制度を整えることと、日常の対話を増やすことは、どちらが欠けても動かない」 |
| 経営者の想いを社員が動ける仕組みに変える | SSR理論／制約が余白と熱狂を生む（スポーツのメタファー） | 理念ポエムで終わらせない。「仕組みに変換する」という具体の行為まで描写する | 「想いを言葉にするだけでなく、社員が行動で選べる基準に変えることが必要」 |
| モノサシ・判断基準・基準のズレ | 認知の非対称性が構造的問題を引き起こす／GROWモデルと「任せる設計」の同型性 | 「正解を押し付ける」ではなく「お互いの判断基準を言語化する」方向で描写する | 「上司と部下でモノサシが違うと、任せても戻ってくる」 |
| 任せる・任せられない | GROWモデルと「任せる設計」の同型性／リーダーシップ（SL理論｜相手の習熟度で関わりを変える） | 「丸投げ」と混同しない。習熟度に応じた関わりの設計が任せることの中身 | 「任せるとは、相手の力に合わせて関わり方を変え続けること」 |
| 評価・評価制度 | 評価の三目的（処遇・育成・活用）／フィードバックの型（事実→影響→対話）／評価運用のパワーバランス | 「査定だけ」「点数をつけること」に見せない。処遇・育成・活用の3目的があり、特に育成・承認の文脈で書く | 「評価は点をつける作業ではなく、何ができて何を伸ばすかを一緒に確認する対話」 |
| 月曜日が待ち遠しい組織・働きたくなる環境 | 動機づけ（自己決定理論SDTと二要因）／選択理論心理学（内側から動く｜5つの欲求と全行動）／SSR理論 | きれいごと・理想論に見せない。「内側から動きたくなる条件を整える」という設計の話として描写する | 「モチベーションを上げるのではなく、モチベーションが下がらない環境をつくる」 |
| 正しいことを言っても動かない | 選択理論心理学／動機づけ（自己決定理論SDTと二要因）／行動経済学（システム1・2と主要バイアス） | 「社員の意識が低い」に滑らない。人は正しさだけでは動かない構造の話にする | 「正しい情報より、動きやすい場面の設計の方が、行動につながりやすい」 |
| 変化への抵抗・なかなか変わらない | 行動経済学（システム1・2と主要バイアス）（現状維持バイアス・損失回避）／システム思考 | 「社員が保守的」に見せない。変化のコストを無意識に大きく感じる認知の仕組みとして描写する | 「変わらないのではなく、変わるための負荷が見えにくい設計になっている」 |
| 伝えても届かない・翻訳 | 認知の非対称性が構造的問題を引き起こす／社会構成主義（対話が現実をつくる） | 「伝え方が悪い」「聞く力がない」に滑らない。経営者と現場の時間軸・判断基準のズレ構造として描写する | 「伝えたは届いたではない。受け取る側の言葉に変換するプロセスが必要」 |
| 承認・認める | 動機づけ（自己決定理論SDTと二要因）／フィードバックの型（事実→影響→対話） | 「褒める」と混同しない。存在・経過・結果の3種類があり、事実ベースで伝えることが核心 | 「承認は褒めることではなく、見ていることを事実で伝えること」 |

## 底流フレーム（本文では用語を出さない）

ブログ本文には名前を出さないが、記事の骨格として意識するフレーム。

| 底流フレーム | frameworks対応エントリ | 本文での表現 |
|---|---|---|
| 社会構成主義 | 社会構成主義（対話が現実をつくる） | 「日々の会話が組織の空気をつくり続けている」 |
| 成人発達理論 | 成人発達理論（垂直的成長｜見方の枠組みが段階的に変わる） | 「幹部が伝える段階から翻訳する段階へ移る」 |
| 選択理論心理学 | 選択理論心理学（内側から動く｜5つの欲求と全行動） | 「人が仕事の中で求めている5つのもの」 |
| 構造原型 | 構造原型（システム・アーキタイプ） | 「同じ問題が繰り返される理由は構造にある」 |
| 公平理論 | 動機の古典（期待理論・公平理論・フロー） | 「給与の納得感は、金額だけでなく、なぜその金額なのかを説明できるかで変わる」 |

## 運用メモ

- この表は全記事の分類表ではなく、ブレ止めの語彙基準。使う言葉が出てきたときに確認する。
- 「外向きの言い換え例」は記事にそのまま貼らない。あくまで方向性の参考。
- 表に載っていない言葉でフレームとの接続が曖昧なときは、`grow3-frameworks.md` の該当エントリを確認してから書く。
- 禁止語・内部用語・文体ルールは `グロウスリー_ブログ執筆コンテキスト_v3.md` 第8節を参照。
- エントリを追加するタイミング：記事を書いていて「この言葉の使い方が毎回ブレている」と感じたとき。
