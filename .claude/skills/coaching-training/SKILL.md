# Grow3 研修採点スキル（座学偏重／ワーク設計の弱さ／1メッセージのぼやけ）

> **適用制約（必読）**：本スキルは参加型・行動変容型の研修（コーチング、ハラスメント、マネジメント、ビジネスマナー等）を対象とする。すべての研修に機械的に適用しない。研修タイプの判定および評価軸の選択は、先に `grow3-judgment` スキル（`references/training-review.md`）を参照すること。知識習得・法令解説・制度説明を主目的とする研修（管理会計研修、法改正セミナー等）では、本スキルの3軸をそのまま適用せず、grow3-judgment側の汎用5軸を用いる。

> 出自：2026年コーチング研修改定（52点→82点）で実際に使った診断軸を汎用化したもの。
> 位置づけ：`grow3-training-design-os.md` 第6部が参照する `grow3-training-scoring-axes.md`（研修採点軸）の「スライドデッキ評価」枠のうち3軸に相当する。GitHub側との同期は運用メモ参照。
> コーチング固有の中身（定義・型）は `grow3-coaching-training-os.md` を参照。本スキルは中身ではなく「構造の良し悪し」を見る。

## このスキルを使うとき

- 研修・ワークショップのPPTX、プログラム案、タイムスケジュールを見て「良し悪しを判定してほしい」と言われたとき
- 「講義が多い気がする」「ワーク中心に変えたい」「内容を絞りたい」という相談を受けたとき
- 新規研修の設計中に、完成前でも構造チェックをしたいとき
- 採点は3軸× 20点 = 60点満点をベースとする。案件固有2軸は原則として追加し、素材の実際の課題に合わせて選定する（例：時間的現実性、現場行動転移設計）。追加しない場合は、その理由を宣言したうえで3軸60点満点で採点してよい。追加する場合は `grow3-training-design-os.md` 第6部の「各軸20点×5＝100点」の慣習に合わせ100点にする

## 採点前の原則（小林さんの既存スタイルに準拠）

- 賛同・同調から入らず、採点→改善FBの順で進める
- 使う評価軸は採点前に必ず宣言する
- 指摘は代替案とセットで出す。代替案なしの指摘はしない
- 判定に必要な情報（時間配分・受講者属性・章立て等）が不足している場合は、採点前に質問する
- 情報不足で判定できない軸は「未確認」として保留し、0点扱いにしない

# 軸1：座学偏重（Lecture-heavy）／20点

**診断の問い**：講義とワークの時間比率はどうか。連続した講義ブロックはどこにあるか。

**判定基準（何をもって座学偏重と判断するか）**

| チェック項目 | 黄色信号 | 赤信号 |
|---|---|---|
| 講義：ワークの時間比率 | 55:45〜60:40 | 65:35以上 |
| 講義が連続する時間 | 15〜20分連続 | 20分超連続 |
| 1つの章に積み上がる理論・概念の数 | 3〜4個 | 5個以上 |
| 「参考」「補足」扱いのスライドの本編混入 | 1〜2枚 | 3枚以上 |

Grow3の目標値は `grow3-training-design-os.md` 第1部の哲学（ワーク比率50%〜55%以上、体験→名前→実験）に準拠する。

**⚠️例外規定（量ではなく接続度で判断する）**

理論・概念の説明は、分量だけで座学偏重と判定してはいけない。判定前に必ず問うこと：**この理論は、研修の中核メッセージ（なぜ今このテーマが必要か）に直結しているか、それとも一般知識として並んでいるだけか。**

直結している理論は、講義時間が長くても減点対象にしない。むしろ研修の説得力の土台であり、圧縮すると中核メッセージが弱まる。

判定の目安：「この理論を削ったら、研修の『なぜ』の部分が弱くなるか？」に「はい」なら残す。「いいえ、他の理論でも代替できる／なくても成立する」なら圧縮候補。

**実例（2026年コーチング研修・重要な誤判定）**：改定検討時、モチベーション理論（マズロー・ハーズバーグ）を「情報量が多い」という理由だけで圧縮候補に挙げたが、実際は「給与は衛生要因に過ぎず、賃上げ環境下でも差がつきにくい構造だからこそ促進要因＝コーチングが重要」という、研修の存在理由そのものに直結する説明だった。実施後、受講者に最も刺さった箇所の一つと判明。分量だけを見て中核メッセージへの接続を確認しなかったことが誤判定の原因。**理論を圧縮候補に挙げる前に、その理論が研修の「なぜ」に接続されているかを必ず確認すること。**

**配点の目安（20点満点）**

- 18〜20点：講義連続が15分以内に収まり、ワーク比率50%以上（ただし中核メッセージに直結する理論説明は講義時間に含めても減点しない）
- 13〜17点：講義連続が15〜20分の箇所が1〜2つあるが、全体では40%台のワーク比率を確保
- 8〜12点：講義連続20分超の箇所が複数あり、ワーク比率が40%を切る
- 0〜7点：ほぼ講義で構成され、ワークが体験というより確認作業になっている

**標準的な打ち手パターン**

1. **章横断で圧縮する（単一章を深追いしない）**：1つの章だけを削っても浮く時間は小さい（実績：単一章では5〜10分、複数章横断だと合計30分規模）。圧縮は必ず2〜3章にまたがって設計する。
2. **理論説明は、中核メッセージへの接続度で圧縮判断する**：接続度が低い理論（一般知識として並んでいるだけのもの）は「概要1〜2枚＋ワークへの導線」に圧縮。接続度が高い理論（研修の「なぜ」を支えるもの）は分量を維持し、むしろ具体的なエピソードで補強する。
3. **章の切り替わり地点に短い口頭問いかけを挟む**：新しいスライドを追加しなくても、2〜3分の自分ごと化の問いを口頭で挟むだけで、連続講義の切れ目を作れる（コスト最小の打ち手）。
4. **参考スライドは配付資料化し、投影から外す**：本編に残す理由がない限り、原則として口頭でも触れない。

# 軸2：ワーク設計の弱さ（Weak exercise design）／20点

**診断の問い**：ワークは「型を知る→小さく試す→本番」の三段階になっているか。いきなり本番のロールプレイに入っていないか。ワークの後に振り返り・共有・転用の設計があるか。

**判定基準**

| チェック項目 | 黄色信号 | 赤信号 |
|---|---|---|
| 本番ワーク（ロールプレイ等）の直前にミニ練習があるか | ミニ練習が講義内の例示のみ | ミニ練習が存在せずいきなり本番 |
| 観察者・オブザーバー役がいる場合、その視点を共有する時間があるか | 各グループ内で留まる | 存在しない |
| 同種のワーク（同じ負荷・同じ形式）が連続していないか | 2つ連続 | 3つ以上連続 |
| ワーク後に「現場でどう使うか」を決める時間があるか | 触れる程度 | 存在しない（学んで終わり） |
| 発散と収束が同じワーク内で混ざっていないか | 一部混在 | 常に混在 |

**配点の目安（20点満点）**

- 18〜20点：型を知る→試す→本番の三段階が揃い、観察者フィードバックと現場実践への接続もある
- 13〜17点：本番ワークはあるが、直前のミニ練習か事後の現場接続のどちらかが欠けている
- 8〜12点：本番ワークのみで、練習も現場接続もない。または同種ワークが3つ以上連続し負荷が偏っている
- 0〜7点：ワークが名ばかりで、実質は講義の確認作業になっている

**標準的な打ち手パターン（学習設計5段階）**

講義 → ミニ練習 → ロールプレイ（本番形式） → 観察者フィードバック → 現場実践計画

1. **ミニ練習を本番の直前に挿入**：本番が15分ロールプレイなら、その前に5分程度「型を短く試す」ワークを置く。苦手な人がその場で固まる事故を防ぐ。
2. **オブザーバー役を作った場合は必ずフィードバック共有の時間を確保**：観察者を置くだけで終わらせず、「何が効いていたか」を全体または他グループと共有する時間を独立して設ける。
3. **現場実践計画を最後に必ず入れる**：相手・タイミング・使うスキル・最初の一手までを具体的に書かせ、ペアで宣言させる。抽象的な感想（頑張ります等）で終わらせない。
4. **同種ワークが連続する場合は統合するか間隔を空ける**：似た負荷のワークが続くときは、片方を削るか、間に異なる種類のワーク・講義を挟んで負荷を分散する。
5. **発散ワークと収束ワークは分けて設計する**：アイデアを広げるフェーズと、絞り込むフェーズを同じ指示文の中で同時に求めない。

# 軸3：1メッセージのぼやけ（Blurred core message）／20点

**診断の問い**：この研修が終わったとき、受講者が一言で言える持ち帰りメッセージは何か。複数のテーマが並立していないか。

**判定基準**

| チェック項目 | 黄色信号 | 赤信号 |
|---|---|---|
| 章立てに含まれるテーマ数（中核テーマ以外） | 1つ隣接テーマが混在 | 2つ以上の独立したテーマが混在 |
| 募集ページ・プログラム表記の項目数 | 4〜5項目 | 6項目以上、かつ相互に関連が薄い |
| 各章の振り返り設問が使い回しか | 一部使い回し | 全章共通・抽象的 |
| 最終章が中核テーマから逸れていないか | やや逸れる | 別テーマ（隣接研修領域）に事実上変わっている |

**配点の目安（20点満点）**

- 18〜20点：全章が一貫して中核テーマに接続し、募集ページの項目も絞られている
- 13〜17点：中核テーマは通っているが、1章分だけ隣接テーマ（別研修で扱うべき内容）が重い
- 8〜12点：複数章にまたがって隣接テーマが混入し、受講者が「結局何の研修だったか」に迷う可能性がある
- 0〜7点：中核テーマが不明瞭で、複数の独立した研修が1つに同居している状態

**標準的な打ち手パターン**

1. **最終章・締め章を中核テーマに強制的に絞り戻す**：最後に隣接テーマ（目標管理、評価制度など）が重くなりがちなので、「今日学んだ中核スキルをどこで使うか」に絞り、隣接テーマは「深掘りは別テーマ」と明言して1〜2枚に留める。
2. **各章の振り返り設問を章ごとに書き分ける**：全章共通の抽象的な設問（印象に残ったことは？等）は形骸化する。各章の中核テーマに直結する具体的な一問（Q1）に差し替え、行動系の設問（変化は？次の行動は？）だけを共通で残す。
3. **募集ページのプログラム項目を「動詞」で書き直す**：名詞の羅列（○○理論、○○モデル）は総花的に見える。「○○を体験する」「○○を練習する」という動詞ベースに直すと、1メッセージが伝わりやすくなる。
4. **削るか残すか迷うテーマは「深掘りしない」と割り切って触れるだけにする**：完全削除ではなく、1〜2枚で言及に留め、詳しくは別テーマとして切り離す判断をする。

# 使い方（実施フロー）

1. 対象の資料（PPTX・プログラム案・タイムスケジュール等）を確認する
2. 3軸を宣言してから採点する（例：「座学偏重・ワーク設計の弱さ・1メッセージのぼやけの3軸、各20点で採点します」）
3. 各軸について、判定基準の該当項目を具体的に指摘したうえで得点を出す
4. 各軸の指摘には必ず打ち手パターンから対応する改善案をセットで示す
5. テーマ固有の事情（時間的制約、対象者の特性等）で判断が変わる場合は、その旨を明示して軸の重みを調整してよい（ただし調整した場合はその理由を明示する）
6. 案件固有2軸は原則として追加する。本スキルの3軸（60点）に加え、素材の実際の課題に合わせて選定した2軸（例：時間的現実性、現場行動転移設計等）を20点ずつ追加し、`grow3-training-design-os.md` の「各軸20点×5＝100点」の慣習に合わせる。追加しない場合は、その理由を宣言したうえで3軸60点満点で採点してよい

## 運用メモ

- 本スキルは構造（座学偏重・ワーク設計・メッセージの絞り込み）を見るための汎用軸。研修の中身の哲学・定義（コーチングとは何か、等）は扱わない。テーマ固有の中身は各テーマの内容OSファイル（例：`grow3-coaching-training-os.md`）を参照する。
- GitHub上の `grow3-training-scoring-axes.md`（研修採点軸）と重複する可能性がある。同ファイルの内容を確認できる場合は、本スキルとの整合を確認し、必要なら統合を検討する。
- 打ち手パターンは2026年コーチング研修改定（52点→82点）での実績ベース。新しい研修で異なるパターンが有効だった場合は、この打ち手一覧に追記していく。

判断に迷ったら references/examples.md の『判断の分かれ目』を参照する。
