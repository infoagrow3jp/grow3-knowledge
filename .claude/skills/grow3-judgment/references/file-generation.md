# PPTX・Word・Excel生成モジュール（ツール手順）

## 着手順序（固定）

1. **対応するスキルファイルを読む**（pptx / docx / xlsx 等）。未読のまま生成を開始しない。
2. **既存資産を確認する**：①キャリアプランニング経由案件か？→YESならCPマスター等の既存テンプレートを確認し踏襲する（運用ルール）。②既存デッキ・文書の改定か？→YESなら編集ワークフロー（editing.md）で対応し、ゼロ作成しない。③ゼロ作成は、既存編集で対応できない新規テーマの初回制作に限る。
3. **構造の骨子を先に確定する**：章立て・時間配分・1メッセージを固めてから中身を作る。
4. pptx-frameスキルのテンプレート資産（content patterns / operational patterns、`.claude/skills/pptx-frame/assets/`）が利用可能な環境では、ゼロ作成前にパターン適用可否を確認する。

## 第0原則との接続

既存ファイルの改定依頼では、指定されたスライド・箇所だけを変更する。レイアウト統一・文言改善など依頼外の変更は本文に入れず、別途提案に回す。

## 出荷前セルフチェック

1. 数値・固有名詞の出所が確認できるか（未確認は【要確認】明記）
2. 配色・体裁がブランド制約に適合するか（`brand-constraints.md`）
3. 公開禁止事例が混入していないか（同上）
4. 受け手が一言で持ち帰れるメッセージが1つに絞れているか

1つでも引っかかれば修正してから提出する。
