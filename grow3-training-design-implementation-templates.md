# Grow3 研修設計 実装テンプレート v1

OS本体（`grow3-training-design-os.md`）から分離した実装用テンプレート集。判断原則はOS本体、概念の正本は`grow3-frameworks.md`を参照。今後、設計チェックリスト・講師ノート標準テンプレート等をここに追加していく。

---

## 暗黙知を引き出す質問（設計ヒアリング用）

熟練者ヒアリング・ケース素材収集で使う。「どう対応しましたか」だけでは結果しか残らない。
判断過程（情報の拾い方と判断基準）を採る。

1. 最初にどこを見たか
2. 何に違和感を持ったか
3. どの時点で判断が変わったか
4. 他にどんな選択肢があったか
5. なぜその選択肢を採らなかったか
6. どこまで自分で判断したか
7. どの条件なら他者へつなぐか
8. 次回も使える判断基準は何か

※組織診断のEvidence採取ヒアリングと同型。診断業務でも使い回せる（`organization-diagnosis_設計メモ_v0.1.md` のEvidence管理を参照）。

## 標準化の仕分け表

暗黙知を形式知化するとき、何でもマニュアルに押し込む事故を防ぐ。仕分けてから、共通標準だけを全員必須にする。

| 分類 | 残す内容 |
| --- | --- |
| 共通標準 | 全員が必ず行うこと（基礎） |
| 判断基準 | 状況に応じて見る観点 |
| 実践例 | 熟練者の工夫や言い方（基礎の上の建物） |
| 例外事例 | 標準どおりでは対応できなかった場面 |
| 実践継承 | 文書では移せず、同行・演習が必要な技能 |
