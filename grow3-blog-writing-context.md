## 4. 記事で使う理論・フレーム（後半の根拠として置く）

### 氷山モデル（4層）

- **できごと**（現象）：離職・評価不満・研修後に戻る
- **時系列パターン**：いつも同じ部署で問題が起きる
- **構造**（仕組み）：評価基準のズレ・役割が言葉になっていない
- **メンタルモデル**（前提）：「どうせ言っても無駄だ」という思い込み

できごとだけを解決してもモグラたたきになる。水面下の構造にアプローチすることで繰り返しを防げる。

### 人が動きたくなる6要因

①意味・目的　②役割・期待値　③成長実感・承認　④関係性・心理的安全　⑤自律・裁量　⑥公正・納得

多くの企業が②⑥だけ整えて「制度を作ったのに変わらない」状態に陥るのは①③④⑤に手をつけていないから。

### 経験学習サイクル

体験→省察→概念化→実験の循環。人は1日後に約70%を忘れる（忘却曲線）。振り返りの仕組みがないと研修は定着しない。

### SSR理論（チームビルディングSSR理論・コーチングファームジャパン）

人材力（強み・特性）→組織力（仕組み・制度）→関係力（対話・信頼）→人材力（さらに深い強みの発見）という好循環サイクル。3つの力は一つが整うと他も動き出す連鎖構造を持つ。

### 経営数値との接続（第4シーズン系記事で使う）

- 2021年以降5年で物価はおよそ13%上昇。賃金を据え置くことは実質的な賃下げ
- MQ（粗利益総額）＝（P単価－V変動費）×Q数量。G（利益）＝MQ－F（固定費）
- 人時生産性＝粗利益÷総労働時間。これが上がると賃上げの原資が生まれる
- 労働分配率＝人件費÷粗利益×100（%）。低すぎると社員への分配が少なく、高すぎると会社に利益が残らない
- 研修・制度・1on1はすべて人時生産性の向上に収束する
- 粗利益が増えれば、同じ労働分配率でも給与水準を上げられる。賃上げを抑える話ではなく、原資をどう生み出すかを考える話

### 時間軸×階層のアラインメント（第4シーズン系記事で使う）

```
長期（3〜5年）  経営者レベル   定性主体（ビジョン・方向性）
中期（1〜2年）  幹部・部門長   定性＋定量
短期（今期・今月） 管理職・現場  定量主体（行動量・数値）
```

- 経営者と現場では見ている時間の長さと仕事の単位が構造的に違う
- 幹部・管理職は「伝令役」ではなく「時間軸を変換する役割」を持つ
- 「伝えた」だけでは届かない理由は伝え方ではなく、この時間軸の違いから来ている

### OKRと評価制度を分ける理由（第4シーズン系記事で使う）

OKRは挑戦・学習・適応のツール。評価制度は公正な貢献の可視化・賃金分配のツール。OKRの達成率を評価や処遇に直接連動させると、社員は安全な目標しか設定しなくなる。

接続の現実解：OKRで挑戦した結果身についた能力・行動を、評価制度の「成長・行動軸」で間接的に評価する。

### 時間軸コンフリクトのマネジメント（第4シーズン系記事で使う）

長期と短期の優先順位がぶつかることは「解消すべき問題」ではなく「マネジメントすべき構造的緊張」。

- 暗黙のまま放置すると現場は常に短期を選ぶ（目の前の仕事は締め切りがあるが、未来の仕事は締め切りが見えにくい）
- ぶつかりを定期的に言葉にする場をつくることで、対立が対話に変わる
- ぶつかりが激しい会社は、組織が成長しようとしているサインかもしれない

### 第5シーズン系の理論・フレーム（底流・表層・実践の3層で使う）

**底流（本文では用語を出さない）：**
- 社会構成主義：「日々の会話や会議、評価面談や1on1の中で、組織の空気は少しずつつくられ続けている」として翻訳する
- 成人発達理論：「幹部が伝える段階から翻訳する段階へ移る」として翻訳する
- 選択心理学：「人が仕事の中で求めているもの」として現場の言葉で表現する

**表層（記事の骨格として正面から扱う。用語は本文末尾に1つまで）：**
- 現状維持バイアス：変化のコストを無意識に大きく感じるメカニズム
- ナッジ・デフォルト効果：動きやすい初期設定の設計
- 損失回避：失うことへの不安が、得ることへの期待より大きく見える
- 確証バイアス：一度持った見方に合う情報を拾いやすくなる
- 承認3分類（存在・経過・結果）：初出時は「存在・経過・結果という3種類の承認」と説明する
- 選択心理学5欲求：人が仕事の中で求めている5つのもの

**実践フレーム（診断・研修導線との接続）：**
- 行動特性3軸（自己表現・自己主張・柔軟性）：shindan.htmlとの接続あり
- ヘルマンモデル：研修設計・チームビルディングの理論根拠として使用

## 5. グロウスリーの立ち位置

**課題設定者としての役割**：AIは問題を解ける時代になった。でもAIは「何を問題と設定するか」ができない。経営者が本当に解きたい問いを一緒に言語化することがグロウスリーの核心的な価値提供。

**ポジティブアプローチ**：問題・欠陥・原因ではなく、理想の状態・強み・仕組みに焦点を当てる。犯人探しではなく、仕組みの改善・環境設計に帰結させる。

**哲学的背景**：「道徳なき経済は犯罪であり、経済なき道徳は寝言である」（二宮尊徳）。利益を生み出す仕組みと、納得して分かち合う評価・賃金を一体で設計することがグロウスリーの思想。

## 6. 記事の基本構造（7ステップ）

すべての記事はこの流れを基本とする。7要素すべてを同じ分量で入れる必要はない。

1. **現場の一言から入る**　経営者が実際に言いそうな一言。抽象論ではなく具体的な場面から始める
2. **受け止める**　いきなり正解を言わない。読者の悩みを否定しない
3. **よくある見方を置く**　「社員の意識が低いのでは」という見方を断罪せずに置く
4. **見方を転換する**　人の問題ではなく、仕組み・構造・対話の問題として捉え直す
5. **水面下の構造に言い換える**　氷山・6要因・SSR等を補助的に使う。理論名は後半に置く
6. **小さな実務に戻す**　「まずは〇〇してみてください」という明日できる一歩
7. **一緒に整理するCTAで終える**　売り込まない。相談したくなる導線

## 7. 文体ルール

### 基本単位
- **1行22文字を目安**（スマホで1行に収まる量）
- 1段落1〜2文。長くなりそうなら改行で区切る
- 1記事1メッセージ。複数のテーマを詰め込まない

### 句読点・記号
- 文末の「。」はつける
- リスト項目の末尾に「。」はつけない
- 「、」を文中に使いすぎない（1文に2個まで目安）
- **改行直前に「、」を置かない**（改行で意味が切れるので不要）
- 体言止めは使ってよいが多用しない

### トーン
- 「先生」ではなく「伴走者」として書く。上から教えるのではなく、一緒に考える
- 読者を「あなた」と呼ばない。「経営者のみなさん」も使わない。場面を描写して共感させる
- 説教調にならない。「べき」より「〜すると動きやすくなります」
- 理論名・フレーム名は共感の後に置く。冒頭に出さない

### 頻出の言い回し統一（v2追加）

| 避ける | 使う |
|---|---|
| 「社員の意識の問題」 | 「設計や場の問題」に転換する |
| 「施策」 | 「打ち手」（第3シーズン以降統一） |
| 「多くの場合こう見られます」 | 「こう考えるのではないでしょうか」 |
| 「構造的に違います」（硬い） | 「見ている時間の長さが違います」「仕事の単位が違います」 |
| 「コンフリクトのマネジメント」 | 「短期と長期のぶつかりを扱う」 |
| 「見える化」 | 「言葉にする」「見えるようにする」 |
| 「渡す」 | 文脈としておかしくなければ「伝える」を優先する |
| 「見る」 | 文脈としておかしくなければ「考える」を優先する |
| 「景色」（原則禁止） | AI文章っぽく見えやすく抽象度が高くなるため文脈に応じて置き換える。「経営者が見ている景色」→「経営者が考えている前提」「経営者の問題意識」／「現場が見ている景色」→「現場が日々向き合っている現実」「現場の仕事の実感」／「同じ景色を見ていない」→「見ている前提が違う」「向き合っている現実が違う」 |

### 見出し・構成
- 見出しは体言止めより問いかけや場面描写が良い
- 小見出しを多用しすぎない。読むリズムを壊さない

### タイトルの基本原則（v4追加・シーズン横断で繰り返された気づき）
- タイトル初稿は理論名・施策名を主語にした説明的・講義口調（例：「〇〇とは何か」「〇〇の重要性」）に寄りやすい。第2・第4シーズンで共通して修正が入った傾向であり、初稿の時点から避ける
- タイトルの主語は「経営者が現場で感じている違和感の言葉」に置く（第3シーズンで確立したルールをシーズン共通原則に格上げ）
- 迷ったら「〇〇していませんか」「なぜ〇〇なのか」など、読者の実感に問いかける形に寄せてから確定する

## 8. 禁止パターン（v2拡充）

### 元からの禁止6パターン

1. **いきなり理論から入る**　「エビングハウスの忘却曲線によると……」→理論は後半に
2. **社員や管理職を責める**　「社員の意識が低いからです」→人ではなく構造へ視点を移す
3. **説教調になる**　「経営者はもっと〜すべきです」→「整えると動きやすくなります」
4. **最後がサービス説明だけになる**　「だから当社の研修を受けてください」→相談導線にする
5. **最後がSSR理論の説明になる**　理論紹介で終わると記事全体が理論解説になる
6. **「だから制度が必要です」と短絡する**　制度を売り込む前に、何を整えるかを一緒に考える流れにする

```

自社ではどこから整えるとよさそうか、
```

### CTA例文

- 自社ではどこから整えるとよさそうか、一緒に言葉にしてみませんか。

## 10. 個別ルール・注意事項

**A4一枚評価制度®**
榎本あつし氏（人事制度の学校・MillReef）のメソッド。初出は®をつける。ブログでは「業績向上と人材育成につながる行動を現場で増やすための共通言語」として表現する。「賃金と必ず連動させる制度」「公平性・納得性を高める制度」とは書かない。

**1on1の定義**
業務進捗支援・内省支援・コンディション支援の3つ。雑談や本音引き出しの場ではない。「経験を振り返り、学びを言葉にし、次の行動につなげる」と表現する。

**SSR理論の帰属**
「チームビルディングSSR理論（株式会社コーチングファームジャパン）の考え方を活用しています」と明記する。「グロウスリー独自のSSR理論」にならないよう注意。

**禁止表現**
- 「社外人事パートナー」→「組織づくりパートナー」
- 「えこひいき」→原則使わない
- 「A4一枚人事制度」→「A4一枚評価制度®」（正式名称）

**経営数値の表現（v2追加）**
- 物価上昇は「13%以上」でなく「およそ13%」（根拠：総務省CPI、2021年平均→2026年3月）
- 人時生産性は定義を明記してから使う（「粗利益÷総労働時間」）
- 「労働分配率を下げる話ではなく、粗利益を増やすことで賃上げの原資をつくる話」として文脈を整える

## 13. 第5シーズン執筆後の文体・構成ルール（v3追加）

### 冒頭定型の連続使用を避ける

以下の型はグロウスリーらしいが、連続すると定型化して見えるため多用しすぎない。

- 「こういう言葉を聞くとき、〇〇に気づくことがあります」
- 「〇〇を考えるきっかけになります」

同じ冒頭パターンは目安として3話に1回程度までに抑える。

**言い換え例：**
- そう感じることがあります。
- 問い直したくなります。
- 〇〇を考え直す場面があります。
- 〇〇の前提を見直した方がよい場面があります。
- そこには別の問題が隠れていることがあります。

### 「見る／見える」は機械的に避けない

「見る」「見える」はAIっぽさや抽象感が出やすいが、機械的に避けると不自然な文章になる。認知や解釈の話では「見方」は使用してよい。

**避けたい表現と置き換え候補：**

| 避けたい表現 | 置き換え候補 |
|---|---|
| 景色 | 前提・現実・立場・問題意識・受け止め方 |
| 〜が見えてくる | 〜に気づく・〜を考えるきっかけになる・〜が浮かび上がる |
| 何でも「見える」で処理する表現 | 感じる・受け取る・拾う・考える |

ただし、不自然な置き換えになる場合は自然な表現を優先する。

### 「一つは／もう一つは」構成を連続させない

打ち手を整理するときに便利だが、連続すると研修資料的になる。

**代替パターン：**
- 「そこには二つのわからないがあります」→問題を先に示して打ち手を後から出す
- 「丁寧に描く3つ＋短く添える3つ」の配分
- 「問いを変える→見方が変わる→関わり方が変わる」という流れ
- 「〇〇ではなく、□□として捉え直す」という転換構文

### ★回の基本型（明文化）

★回では理論を正面から扱うが、理論説明を主役にしない。

1. 現場の違和感・経営者の言葉から入る
2. 個人の性格・意欲・能力の問題にしない
3. 認知・心理・対話・制度・場の設計に置き換える
4. 理論名は本文末尾に1回だけ出す
5. 最後は「自社で何を問い直すか」に戻す

理論説明は原則2〜3文以内に抑える。理論は読者を説得するためではなく、日常の違和感に名前を与えるために使う。

### CTAの安心文は重ねすぎない

安心させる文は1つで十分。その代わり、読者の実際の相談言葉を入れる。

**NG例（重複）：**
- 「どこから手をつければいいかわからなくても大丈夫です」
- 「仕組みの話としてまとまっていなくても大丈夫です」

**推奨例：**
安心文は1つに絞り、読者の言葉を添える。
「うちの組織、こういう状態なんですが」「何から始めればいいかわからなくて」そのままの言葉で話してもらえればそこから一緒に考える、という流れ。

### 良い記事は、読後に新しい問いが残る

グロウスリーのブログは、答えを渡す記事ではなく、問いを変える記事である。

記事の最後では、読者が自社の問題を別の角度から考え直せる問いを残す。

**問いの転換の例：**
- 社員が悪いのか、動きにくい場になっているのか。
- 信頼が足りないのか、信頼を支える情報が足りないのか。
- 変わっていないのか、変化のサインを受け取れていないのか。
- 完成していないのか、更新し続けている途中なのか。

### 採点時の補助観点（v3追加）

既存の採点5軸に加えて以下を確認する。

1. **前後の記事との差別化**：似たテーマが続く場合、今回の記事固有の役割が立っているか
2. **理論説明が研修資料的になっていないか**：理論名が主役になっていないか
3. **カテゴリーの性質に合っているか**
   - 「現場に根づく実践」：具体例の密度を高める
   - 「経営者の悩みをほどく」：自責や苛立ちを、情報・判断基準・見方・対話の設計に置き換える
   - 「人事制度・評価・賃金」：制度の正しさだけでなく、社員がどう受け取りどんな行動につながるかまで扱う
   - 「組織づくりの考え方」：思想を語りつつ、日常の場面に戻す

## 14. ステップメール執筆ルール（v3追加）

ステップメールはブログと同じ文体思想を使うが、役割が異なる。

ブログは1記事で問いを立て、読者に考える余白を残す。  
ステップメールは、複数通を通じて読者の見方を少しずつ変えていく。

### 基本原則

- 1通1メッセージ・1通1問いに絞る
- ブログより短く、解説しすぎない
- 理論名は原則出さない
- 施策説明ではなく、経営者の見方が変わる流れをつくる
- 感謝や挨拶より、読者の内側の言葉から入る
- 最終通以外は売り込みを強くしない
- 最終通は問いで終えず、相談への行動導線を置く

### 基本構成

1. **読者の内側の言葉から入る**  
   例：「何度伝えても、なぜ変わらないんだろう」

2. **よくある見方を置く**  
   例：「人の問題なのか」と考えたくなることがあります。

3. **問いを転換する**  
   例：「人が悪い」のではなく「動きにくい場になっている」のかもしれない。

4. **小さな具体例を出す**  
   例：相談のタイミング、1on1シート、評価面談、会議での問い。

5. **1つだけ持ち帰る問いを置く**  
   例：社員が変わらないのか。それとも、動きにくい場になっているのか。

6. **次回導線を置く**  
   次回予告は、原則として確定件名をそのまま使う。

### 書き出しのルール

メルマガ的な挨拶から入りすぎない。

**避けたい例：**
- メールを受け取ってくださってありがとうございます。
- 7回にわたってお届けしてきた最後のメールです。

**使いやすい例：**
- 「相談した方がいいのかもしれない。でも、何を話せばいいかわからない」
- 「何度伝えても、なぜ変わらないんだろう」
- 「変わってきているんでしょうか。正直、よくわからなくて」

### 締め方のルール

毎回同じ締め方にしない。

**使い分け例：**
- 今日、考えてみたい問いは一つだけです。
- まずは、この問いからで十分です。
- ここで一度、見直してみたい問いがあります。
- 今回、考えてみたいのはこの一点です。

最終通は問いで終えるのではなく、相談への行動導線を置く。

### CTAルール

1〜6通目は、次回導線を中心にする。  
7通目で初回相談への導線を明確にする。

**最終通の中心文：**
```
相談は
答えを持っていく場ではありません。

一緒に状況を整理する時間です。
```

**安心文は重ねすぎない：**
```
仕組みの話として
まとまっていなくても大丈夫です。

気になっていることをそのまま
話すところから始められます。
```

**リンク表記：**
```
初回相談はこちら（無料）
https://grow3.jp/contact2/
```

### 確定ステップメール件名（7通）

| 通 | 件名 |
|---|---|
| 1 | 「何度伝えても変わらない」は、人の問題ではないかもしれない |
| 2 | 動かない社員の前に、動きにくい場がある |
| 3 | 社長の言葉は、なぜ現場で別の意味になるのか |
| 4 | いい制度なのに使われない会社で起きていること |
| 5 | 社員が動き始めた会社に増えるサイン |
| 6 | 「いつになったら整うのか」と焦っている経営者へ |
| 7 | 相談は、整理できてからでなくていい |
