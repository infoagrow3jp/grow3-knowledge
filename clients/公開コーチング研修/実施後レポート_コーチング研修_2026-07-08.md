# コーチング研修 実施後レポート

株式会社グロウスリー　小林裕司
実施日：2026年7月8日（水）／対象：公開研修・受講者9社12名／時間：10:00〜17:00

## 1. 実施概要と研修目的

管理職が「答えを渡す」だけでは部下の判断力と行動が育ちにくく、かつ給与差がつきにくい環境では、達成・承認・成長実感といった促進要因を引き出す関わりがより重要になっています。本研修は、コーチングを「相手を変える技術」ではなく、「相手が自分で考え、判断し、動ける状態を一緒につくる対話」として捉え直し、1on1や日常面談で使える聴き方・問いかけ方の型を練習することを目的としました。

当日は、なぜ今コーチングが必要か（背景）→ティーチングとコーチングの使い分け→承認・傾聴・質問・フィードバックの関わり方→現場での使いどころ、という流れで構成しました。講義のあとミニ練習・実践ワーク・現場実践計画までつなぎ、「知って終わり」にならない設計としました。

事前に合意した到達イメージは、受講者が「部下に答えを出す場面」と「引き出す場面」を見分け、承認・傾聴・質問のいずれかを翌週の会話で試せる状態まで進むことです。

## 2. 受講者の反応・理解が進んだ点

（以下は、実施後の振り返りで確定した観察事実です。受講後アンケートはキャリアプランニング側が回収しており、集計値・自由記述は本時点で未入手のため定量結果は記載していません。【要確認】）

観察できた反応として、次の点が特に理解の前進につながったと見られます。

- **コーチングの二層構造**（論点はコーチが握り、答えの評価は握らない）の説明に、複数の受講者から強い反応がありました。方法論だけを「質問テクニック」として捉える誤解がほどけ、対話の設計として理解が進んだ場面でした。
- 当日の対話の中で生まれた表現として、「前提や背景を固めずに相手に答えがあると信じるのは危うい」という指摘への共感が広がりました。コーチングを楽観的な信念ではなく、状況を見極めた使い分けとして受け止めるきっかけになったと観察しています。
- 「相手に答えがない場合はティーチングに切り替えてよい」という整理に対し、安堵の反応が見られました。コーチング万能論への負担が下がり、現場で使える判断基準として受け取られたと見ています。
- **承認シャワー**は好評で、「職場でもやってみたい」という声がありました。一方で、「職場はオープンマインドではないので厳しいかも」という本音も出ており、研修内の体験と職場実装のギャップがすでに言語化されています。
- 質問例のミニ練習、ヒーローインタビュー、コーチング実践、現場実践計画についても、概ね前向きな取り組みが見られました（詳細な発言ログは未確認）。

## 3. 現場定着に向けた期待と懸念

**見立て（定着しやすい点）**
- 二層構造とティーチング／コーチングの切り替えは、観察上もっとも腹落ちした論点です。翌週の1on1で「今は教える／今は引き出す」を一言決めてから会話に入る使い方が、定着しやすいと見ています。
- 現場実践計画まで記入・宣言できたため、「誰に・いつ・どのスキルを使うか」が個人単位で具体化されています。計画の有無は定着の初期条件として有利です。

**懸念（定着を妨げるリスク）**
- **職場要因**：承認の練習は研修内では機能しましたが、「職場がオープンマインドでない」という声が出ています。上司・同僚の反応が冷たいと、承認・傾聴の試行が早期に止まりやすいです。
- **時間要因**：当日は学びの言語化を個人記入中心の構成としたため、共有の場は職場側で設けていただく設計です。会社側で共有の場がないと、個人の気づきが組織知にならず薄れやすいです。
- **スキル要因**：コーチング実践はペア形式で回数を確保しましたが、研修内で確保できる反復回数には限りがあるため、問いの質（現状把握の深掘りなど）は、職場での反復練習がないと戻りやすいです。
- **仕組み要因**：評価面談・週次1on1・朝礼など、既存の会話枠に接続されないと、実践計画が「書いて終わり」になりやすいです。

※行動変容の有無は本時点では断定できません。上記は当日観察に基づく見立てです。

## 4. 管理者・会社側に必要な支援

参加各社の管理職・人事の方には、次の支援をお願いします（今月〜来月の行動レベル）。

1. **週次1on1で「教える／引き出す」を1回だけ意識する**  
   面談の冒頭30秒で、「今日はこのテーマについて引き出す」または「今日は手順を教える」と決めてから話す。
2. **承認を「場のイベント」ではなく1対1の一言に落とす**  
   グループでの承認体験をそのまま職場に持ち込まず、日常会話で I／You／We のいずれか一言を添える形から始める（研修内でも職場実装の難しさが指摘されています）。
3. **現場実践計画の実行日をカレンダーに入れる**  
   受講者本人が書いた「相手・日時・使うスキル・最初の問い」を、上司または人事が1回だけ確認し、実施後に「どうだったか」を5分聞く。
4. **失敗しても話してよい空気を先に宣言する**  
   「うまく聞けなくてもよい。まず1回試す」ことをチーム内で共有し、オープンマインド不足への不安を下げる。
