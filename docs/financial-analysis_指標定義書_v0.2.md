# financial-analysis 指標定義書 v0.2

本書は financial-analysis（財務分析サブエージェント）が使用する指標の定義・算式・
符号規約・区分ルールの正本である。計算が正確でも定義が揺れれば意味の違う数字を
比較する事故が起きるため、実装（計算スクリプト・スキル・エージェント）は本書に
従属する。本書にない定義を実装側で創作してはならない。

## 0. 共通原則

1. **暗算禁止**：すべての計算はコード実行で行い、入力値・使用式・丸め前数値・
   使用定義を保存する。
2. **算式併記義務**：レポートに指標を出すときは必ず「指標名：値（算式：分子÷分母、
   使用定義ID）」まで表示する。
   例「債務償還年数：8.4年（算式：要償還債務（JFC定義）÷簡易営業CF）」。
3. **根拠区分**：閾値・目安には必ず次のいずれかのタグを付ける。
   official_criterion（公的制度・再生計画等）／statistical_benchmark（同業・
   同規模統計）／financial_institution_view（金融機関実務の見方）／
   grow3_heuristic（Grow3内部経験則）／client_target（当該企業の目標）。
4. **確からしさグレード**：各判定に A（確定計算）／B（推定・補正込み）／
   C（要確認）を付す。判定できない指標は0点扱いせず「算定不能（理由）」とする。
5. **符号規約**：キャッシュフローはすべて「入金＝正、出金＝負」で保持する。
   「投資CF」はCF計算書の負数のまま扱う。支出額を正数で扱う場合は
   「設備投資支出額」と別名で呼び、混用しない。
6. **分析単位の宣言**：MQ系の計算前に、Qの単位（個数／件数／時間等）・期間
   （月次／年間）・範囲（全社／商品別／工事別）を宣言してから計算する。
7. **定義変更時の検算必須**：本書の定義を変更・追加するときは、数式・比率・
   方向性を含む定義について、最小の数値例（例：P=100、V=60、Q=10）で一度
   計算し、次元（単位）と方向（増減の向き）を確認してから記載する。
   文章だけで定義を確定させない。

## 1. キャッシュフロー系指標（4定義を厳密に区別）

| 名称 | 定義 | 取得元 | 備考 |
|---|---|---|---|
| 実績営業CF | CF計算書の営業活動によるCF | CF計算書 | 無い場合はB/S増減から運転資本変動込みで再構成（§9参照）。再構成資料もなければ算定不能 |
| 簡易営業CF | 経常利益＋減価償却費−法人税等 | P/L・内訳書 | **FCFと呼ばない**（簡便法であり運転資本増減を含まない）。実態修正（§8）を行った場合は補正後経常利益から算出し「補正後簡易営業CF」と表示 |
| 広義FCF | 実績営業CF＋投資CF（投資CFは負数のまま加算） | CF計算書 | 投資CFには設備投資のほか投融資・M&A・資産売却等も含むため、本業の投資後CFや返済余力とは一致しない場合がある。単に「FCF」と呼ばない |
| 返済原資CF | 補正後簡易営業CF−維持更新に必要な設備投資−P/Lに反映されていない事業外資金流出 | 計算＋個社判断 | **計算値＋調整項目＋調整理由の3列**で扱う。自動確定しない。役員報酬の過大過小は§8の利益補正側で処理し、ここで再控除しない（二重控除禁止） |

## 2. 債務償還年数と返済負担

**分子（debt_numerator_definition_id を必ず付す）**：
- ①総有利子負債
- ②ネット有利子負債（総有利子負債−現預金）
- ③出典定義による要償還債務（例：日本政策金融公庫・経営改善計画の定義。使用した
  出典と定義を明記）
- ④Grow3調整後要返済債務（有利子負債−正常運転資金−**余剰現預金**。現預金は全額
  控除せず、事業継続に必要な手元資金を除いた余剰分のみ控除。正常運転資金＝
  売掛債権＋棚卸資産−仕入債務）

**分母**：①簡易営業CF（一次スクリーニングの標準）／②実績営業CF

- **標準採用**：一次スクリーニングは「①÷簡易営業CF」、精査時に「④÷簡易営業CF」
  を併記。定義の異なる年数どうしを比較しない。
- 閾値：10年・20年は絶対基準ではない。「概ね10年以内」＝official_criterion
  （再生支援等における計画終了時の目安）。「10年超で一次注意・20年超で危険」＝
  grow3_heuristic【要確認：小林さん相場観】。業種別許容幅は
  statistical_benchmark確認のうえ設定【要確認】。
- 分母がゼロ・マイナスの場合は年数を算出せず「償還原資不足」と表示し、
  停止条件（§10）に該当させる。

**年間元本返済負担率（新設）**：
- 年間元本返済負担率＝今後12か月の元本返済予定額÷返済原資CF。
  利息まで含める場合は「年間債務返済額÷返済原資CF」として別名で定義する。
- 2026年文脈：ゼロゼロ融資の据置明け・借換えは債務償還年数（残高÷CF）ではなく
  この返済負担率に現れる。借入金返済予定表で内訳（据置・借換・新規）を確認して
  から解釈する。

## 3. 人件費の定義（3層構造）

| 層 | 内容 |
|---|---|
| 雇用人件費 | 役員報酬・給与・賞与・法定福利費・福利厚生費等。**役員込み／従業員のみを常に併記** |
| 外部労働費 | 派遣費・業務委託費・外注費のうち人的役務部分。材料込み外注（建設等）は分解できる場合のみ人的部分を計上、分解不能なら注記のうえ除外 |
| 総人的コスト | 雇用人件費＋外部労働費 |

- 賃上げ余力の計算は法定福利費込みの実効人件費で行う（給与増×(1＋法定福利費率)）。
- 「実質人件費比率」という語は使わず**総人的コスト比率**と呼ぶ。

## 4. 労働分配率系指標（定義IDで厳密に区別・混同禁止）

すべての算定値に次のメタデータを付す：
`statistical_source / definition_id / reference_year / industry_class /
company_size_class`

- **労働分配率（統計定義）**：人件費÷付加価値額。付加価値額・人件費の範囲は
  使用する統計の定義に従う（法人企業統計では賃借料・租税公課等を含む固有の算式。
  ≠売上総利益、≠M）。統計比較はこの指標でのみ行い、**企業規模と使用統計・
  定義IDを揃える**（規模別に水準が大きく異なるため、業種だけの一律レンジを
  使わない）。公的統計の正本選定は【要確認：中小企業実態基本調査／法人企業統計
  等から小林さん選定】。
- **労働分配率（BAST定義）**：人件費÷限界利益×100。BAST用語集の定義で、
  人件費には製造原価中の賃金・賞与・雑給・法定福利費・厚生費・退職金、
  販管費中の従業員給与・役員報酬・賞与・厚生費等を含む（役員報酬・法定福利費
  込み）。**Grow3のMQ雇用人件費比率と概ね対応**し、BAST速報版との比較は
  この定義IDで行う（§6-2参照）。
- **MQ雇用人件費比率＝雇用人件費÷MQ**、**MQ総人的コスト比率＝総人的コスト÷MQ**：
  Grow3の打ち手検討用。分母のMQがゼロ・マイナスの場合は比率を算出せず
  「MQ不足のため算定不能」とする。「労働分配率」とは呼ばない。

## 5. MQ会計の定義とV/F区分

- P（販売単価）・V（単位当たり比例費）・M＝P−V・Q（数量）・MQ＝M×Q・
  F（固定費）・G＝MQ−F。
- **V/F区分ルール（確定）**：
  1. 分析単位・期間を宣言する（§0-6）。
  2. **Qの増減にほぼ比例して増減する費用のみをVとする。それ以外はFとする。**
  3. 迷う科目は**暫定F＋「区分要確認」フラグ**とし、必要に応じてVとした場合の
     感応度を併記する。Fに置く理由は保守性ではなく、比例性を確認できない費用を
     Vと断定しないためである（費用をFへ置くとMは大きくなる。方向を誤らない）。
  4. 個社上書きは可。上書き時は理由と適用期間を記録する。
  5. 段階的に増える費用（一定Qごとに増える人員・設備等）は、将来拡張として
     「段階固定費」の別表示を予定（v0.2では暫定F＋注記）。
- **業種別初期マッピング（製造・建設。すべて「候補」であり比例性を個社実態で
  確認する）**：
  - 製造：**材料消費額**（当期購入額ではなく当該期間の生産・販売に対応する
    消費額）・外注加工費＝**V候補**／運賃は出荷量比例ならV候補・月額固定便はF／
    正社員労務費＝F／日雇・案件単位の応援＝V候補。
  - 建設：材料費・**工事別**外注費＝**V候補**。月額固定契約・常用契約・
    材料や機械込みの外注は内訳を確認してから区分／現場経費は一括分類せず
    内訳確認／常用職人＝F・応援＝V候補。
  - 月額固定の派遣・業務委託＝FまたはF段階（注記）。

## 6. MQミックス最適化とベンチマーク参照

### 6-1. MQミックス最適化

- 順位付けの主指標は m率（M÷P）ではなく **MQ総額**、さらに
  **制約資源あたりMQ**（人時・機械時間等、その会社のボトルネック1単位あたり）。
- 見かけ赤字商品（F配賦後赤字）の廃止検討時は、**その商品のMQを失った場合のG**を
  必ず計算してから判断する。
- 値引き案には「値引きn%を取り返すのに必要なQ増加率」の検算を必須とする。
- 感度分析はP+1%／V−1%／Q+1%／F−1%の4本を並べ、Gへの影響を比較する。
- **Qの需要予測を根拠なく置かない**。仮定はシナリオとして明示し、
  見込みとして断定しない。
- 顧客軸のミックス（値上げ受容度・採算による顧客構成の見直し）も同じ枠組みで扱う。

### 6-2. ベンチマーク参照ルール（保存せず、都度参照する）

- **正本の参照先**：
  1. TKC経営指標（BAST）速報版：https://www.tkc.jp/tkcnf/bast/sample/
     （登録不要・206業種12項目・四半期更新。限界利益率・労働分配率（BAST定義）・
     損益分岐点比率・1人当り限界利益等、MQ管理会計モードのベンチマークとして使用）
  2. 公的統計（労働分配率（統計定義）・債務償還・安全性系）：正本の選定は
     【要確認：小林さん】
- **権利上のルール（厳守）**：TKCサイトの著作権規定は複製・転載・再配布を
  禁止している。したがって**BASTの数表・数値をリポジトリやデータファイルに
  保存・蓄積しない**。分析実行時にWebで参照（都度取得）する。顧問先向け
  レポートへの記載は、出典（「令和◯年版TKC経営指標（BAST）速報版・TKC全国会」）
  と対象決算期を明記のうえ、比較に必要な業種の数値のみ最小限にとどめる。
- **BAST要約版は令和8年8月31日でサービス終了**。要約版に依存する設計をしない。
  速報版にない指標（債務償還能力等）のベンチマークは公的統計・
  financial_institution_view で受け持つ。
- **鮮度と対応**：参照時は必ず対象決算期（例：令和8年1月〜3月決算）を記録する。
  ベンチマークにない業種・規模は**値を創作せず「該当ベンチマークなし・要確認」**
  とする。
- 比較は定義IDを揃えて行う（BAST比較はBAST定義、統計比較は統計定義。混用禁止）。

## 7. 時代前提（仮説として保持）

分析基準日時点の経営環境を確認する。2026年時点はコスト上昇（円安・資源高・
人手不足による賃上げ圧力）を踏まえ、**P戦略を優先的な検討候補とする**が、
企業ごとの競争環境・顧客構成・契約条件・価格弾力性・失注リスクを確認して判断する。
V・F削減を第一提案としない既定は仮説であり、輸出型・下請構造等で反転しうる。
環境前提と内部目安は年1回以上見直す。P改定の実行条件（価値の言語化・顧客構成
シフト・値決めできる組織）は打ち手設計で必ずセットにする。

## 8. 実態修正

- 役員報酬の過大過小／生命保険等による利益調整／役員借入金等を補正した
  実質収益力・実態B/Sを併記する。補正は「計上値・補正値・補正理由」の3列で
  記録し、補正前後を必ず両方示す。CF系指標（§1）は補正後利益から再計算し
  「補正後」を冠して表示する。
- **役員借入金**：返済意思・返済実績・契約期間・劣後性・長期固定性を確認し、
  **内部分析上の資本性評価候補**とする。**法定B/S上の負債区分は変更しない**。
- **減価償却**：固定資産台帳・取得日・使用状況・採用した会計方針・税務上の
  償却限度額・実際計上額を**区別して**比較する（税務上の償却限度額と会計上
  必要な「相当の減価償却」は同一概念ではない）。

## 9. 資産実在性スクリーニング（異常兆候の検知）

**原則**：各シグナルは不正・粉飾の判定ではなく、実在性・回収可能性・時価・
滞留状況を追加確認する契機として扱う。単一指標で判定せず、3期推移と補助資料を
組み合わせる。顧客向け出力では断定せず確認質問に変換する（「粉飾」という語を
顧客向け出力で使わない）。

| シグナル | 見る方法 |
|---|---|
| 繰越利益剰余金マイナス×流動資産（現金・売掛金・商品）の膨張 | B/S3期推移 |
| 売掛債権・棚卸資産の回転期間の急伸 | 売上と回転期間の3期比較 |
| 黒字なのに**実績営業CF**が連続マイナス | P/LとCF計算書の突合。CF計算書がない場合はB/S増減から運転資本変動を含む営業CFを再構成する。**簡易営業CFでは代替しない**（運転資本増減を含まず兆候が見えないため）。再構成資料がなければ算定不能 |
| 現金残高と借入の同時膨張 | B/S・返済予定表 |
| 仮払金・貸付金・役員貸付金等の雑勘定膨張 | 勘定科目内訳明細書 |
| 投資有価証券のウエイトが高い | 時価・保有目的の確認 |
| 減価償却の計上不足の疑い | §8の方法で比較 |

兆候検知時の追加確認資料：売掛金年齢表・回収予定／棚卸資産明細・滞留期間／
預金残高資料／固定資産台帳・償却明細／投資有価証券の時価・保有目的／
仮払金・貸付金等の相手先と発生理由。※最初から全資料は要求しない。

## 10. 工程・出力・停止条件

- **工程3段**：Step1 入力検証（不足を計算可能／参考値／算定不能に分類）→
  Step2 計算（コード実行・§0の保存要件）→ Step3 見立て
  （事実→解釈候補（複数仮説を残す）→追加確認→打ち手候補）。
- **入力資料**：最小3点（3期分B/S・P/L・販管費内訳／勘定科目内訳明細書／
  借入金返済予定表）で基礎分析を開始できる。指標ごとに必要資料を定め、
  不足する指標は算定不能または参考値と表示する。
- **出力2モード**：経営者向け（平易・非断定・確認質問・論点の絞り込み・
  計算過程は最小限）／Grow3内部用（入力値と出典・算式と定義ID・仮区分と補正・
  異常検知フラグ・解釈候補・追加確認資料・小林レビュー要否）。
  銀行向けは第2弾（初期は「金融機関からどう見られる可能性があるか」の
  経営者向け解説にとどめる）。
- **停止条件（要・小林レビュー：該当時は顧客向け出力を確定しない）**：
  ①資産実在性・回収可能性の兆候あり／②債務償還年数が内部危険水準、または
  分母ゼロ・マイナス／③重要科目のV/F区分が未確定／④不足資料・補正値に
  よって結論が変わる可能性がある／⑤**B/Sが貸借一致しない、前期末と当期首が
  一致しない、明細と決算書に重要な不整合がある**（見立て以前の入力エラー）。

## 11. 責任境界・守秘

- 税額計算の断定・個別の節税提案をしない（顧問税理士と連携前提。税額・
  納税資金は概算と明示）。投資判断・融資可否を断定しない（判断材料の提示まで）。
- 顧問先の実数値・実データを公開リポジトリ（grow3-knowledge）に置かない。
  実データはDropbox関与先フォルダで扱う。eval素材・examplesは架空明記を必須とする。
- 外部ベンチマークの権利ルールは§6-2に従う。

## 12. 適用範囲・初期運用（7論点の確定）

   人的サービス業に分けて展開。
